# UX-001 1차 MVP 구독 사용자 경험 설계

## 문서 상태

- 작업 ID: `UX-001`
- 역할: UX/UI Designer
- 설계 상태: Proposed UX Design
- 기준 입력: `docs/product/PS-002-first-mvp-requirements.md`, `docs/domain/DOMAIN-001-first-mvp-subscription-domain.md`
- 대상 사용자: 구독을 생성하고 자신의 구독을 조회하는 로그인 회원

이 문서는 프론트엔드 구현과 QA 검증을 위한 화면·상태 설계다. 사용자 승인 전까지 화면 배치와 이동 방식은 Proposed UX Design 상태로 다룬다.

## 승인된 제품 입력 요약

- 상품 검증 범위는 개·고양이용 사료다.
- 구독 하나는 SKU 하나만 대상으로 한다.
- 회원은 존재하며 구독 가능한 SKU만 구독할 수 있다.
- 수량은 1~10이다.
- 배송 주기는 2주, 4주, 8주다.
- 다음 주문 예정일은 구독 생성일에 선택한 배송 주기를 더한 날짜다.
- 날짜 계산과 표시는 `Asia/Seoul` 기준 날짜 단위로 다룬다.
- 휴일, 주말과 영업일 보정은 적용하지 않는다.
- 화면 날짜 표시는 `YYYY. M. D.`를 사용한다.
- 다음 배송 예정일은 표시하지 않는다.
- 첫 번째 MVP에는 구독 상태, 결제, 재고, 배송, 구독 변경 기능이 없다.
- 비회원이 구독 생성·내 구독 목록·구독 상세에 접근하면 로그인 화면으로 이동한다.
- 다른 회원의 구독은 조회할 수 없는 구독으로 처리한다.
- HTTP 상태, 오류 코드, API 오류 응답 JSON은 이 문서에서 확정하지 않는다.

## 핵심 사용자 흐름

### 상품 탐색에서 구독 생성까지

```text
상품 목록
→ 상품 상세
→ SKU 목록 확인
→ 구독 가능한 SKU 선택
→ 수량 1~10 입력
→ 배송 주기 2주·4주·8주 중 선택
→ 입력 검토
→ 구독 생성
→ 생성 처리 상태
→ 생성 성공
→ 생성된 구독 상세 확인
```

### 내 구독 조회

```text
내 구독 목록
→ 구독 목록 항목 선택
→ 내 구독 상세
→ 상품명, SKU, 수량, 배송 주기, 구독 생성일, 다음 주문 예정일 확인
```

### 제한 접근

```text
비회원이 구독 생성·내 구독 목록·구독 상세 접근
→ 별도 중간 페이지 없이 로그인 화면으로 이동
```

```text
로그인한 회원이 존재하지 않는 구독이나 다른 회원 소유의 구독 상세에 접근
→ 구독을 확인할 수 없는 상태
```

## 화면별 텍스트 와이어프레임

### UX-SCREEN-001 상품 목록

```text
[상단]
- 서비스명 또는 페이지 제목: 상품
- 보조 이동: 내 구독

[본문]
- 화면 제목: 정기배송 가능한 사료 상품
- 상품 카드 목록
  - 상품 ID
  - 상품명
  - 대상 동물: 개 또는 고양이
  - 짧은 설명
  - SKU별 표시 가격 요약
  - 구독 가능한 SKU 있음 / 구독 가능한 SKU 없음

[주요 액션]
- 각 상품 카드: 상세 보기
```

표시 순서는 상품 ID, 상품명, 대상 동물, 짧은 설명, SKU 가격 요약, 구독 가능 여부, 상세 보기 순서다.

Proposed UX Design:

- 구독 가능한 SKU가 하나 이상 있는 상품은 “구독 가능한 옵션 있음” 문구를 표시한다.
- 구독 가능한 SKU가 없는 상품은 “현재 구독 가능한 옵션 없음” 문구를 표시하되, 재고·품절 정책으로 설명하지 않는다.
- 검색, 정렬, 필터, 페이지네이션은 제공하지 않는다.

### UX-SCREEN-002 상품 상세 및 구독 입력

```text
[상단]
- 이전으로
- 상품 상세

[본문: 상품 정보]
- 상품명
- 대상 동물
- 상품 설명

[본문: SKU 선택]
- 섹션 제목: 옵션 선택
- SKU 선택 목록
  - SKU 용량 또는 구성
  - SKU 표시 가격
  - 구독 가능 여부
  - 구독 불가능한 SKU는 선택 불가

[본문: 구독 입력]
- 수량 입력
  - 숫자 입력 또는 스텝퍼
  - 허용 범위 안내: 1~10
- 배송 주기 선택
  - 2주
  - 4주
  - 8주

[본문: 입력 검토]
- 선택한 상품명
- 선택한 SKU
- SKU 표시 가격
- 수량
- 배송 주기
- 안내 문구: 다음 주문 예정일은 구독을 만든 뒤 확인할 수 있습니다.

[주요 액션]
- 구독 만들기
  - 기본 상태: 활성
  - 제출 중에만 비활성화
```

Proposed UX Design:

- 상품 상세 화면 안에서 SKU·수량·배송 주기를 선택하고 구독을 생성한다.
- SKU 선택 전에는 입력 검토 영역에 “옵션을 선택하면 구독 내용을 확인할 수 있습니다.”를 표시한다.
- 입력 검토 영역에는 구체적인 다음 주문 예정일을 미리 계산해 표시하지 않는다.
- 수량과 배송 주기가 선택된 뒤에도 “다음 주문 예정일은 구독을 만든 뒤 확인할 수 있습니다.”를 표시한다.
- `구독 만들기` 버튼은 기본적으로 활성 상태로 둔다.
- `구독 만들기` 실행 시 입력 검증을 먼저 수행하고, 오류가 있으면 구독 생성 요청을 보내지 않는다.
- 오류가 여러 개이면 오류 요약을 표시하고, 포커스는 오류 요약 또는 첫 번째 유효하지 않은 입력으로 이동한다.
- 오류를 수정한 뒤 다시 `구독 만들기`를 실행할 수 있다.
- 다음 배송 예정일은 어떤 위치에도 표시하지 않는다.
- 비회원이 구독 만들기 또는 제한된 구독 입력 영역에 접근하면 로그인 화면으로 이동한다.

### UX-SCREEN-003 구독 생성 처리 상태

```text
[상단]
- 상품 상세

[본문]
- 처리 메시지: 구독을 만들고 있습니다.
- 선택 정보 요약
  - 상품명
  - SKU
  - 수량
  - 배송 주기

[주요 액션]
- 구독 만들기 버튼 제출 중 비활성화
- 중복 제출 방지
```

Proposed UX Design:

- 제출 중에는 SKU, 수량, 배송 주기 입력을 잠근다.
- 같은 요청이 반복 제출되지 않도록 제출 중에만 주요 액션 버튼을 비활성화한다.
- 처리 중 상태는 보조기기에 알린다.

### UX-SCREEN-004 구독 생성 성공 결과 또는 구독 상세

```text
[상단]
- 구독 생성 완료

[본문]
- 성공 메시지: 구독이 만들어졌습니다.
- 구독 ID
- 상품명
- SKU 용량 또는 구성
- SKU 표시 가격
- 수량
- 배송 주기
- 구독 생성일
- 다음 주문 예정일

[주요 액션]
- 내 구독 상세 보기
- 내 구독 목록 보기
- 상품 목록으로 이동
```

Proposed UX Design:

- 구독 생성 성공 후 생성된 구독 상세 화면으로 이동하는 방식을 기본안으로 제안한다.
- 기술 또는 라우팅 제약이 있으면 동일한 정보 구조의 성공 결과 화면을 먼저 보여주고, “내 구독 상세 보기”를 제공해도 된다.
- 어느 방식이든 생성 직후 구독 ID와 다음 주문 예정일을 확인할 수 있어야 한다.

### UX-SCREEN-005 내 구독 목록

```text
[상단]
- 내 구독
- 보조 이동: 상품 목록

[본문]
- 화면 제목: 내 구독
- 구독 목록
  - 구독 ID
  - 상품명
  - SKU 용량 또는 구성
  - 수량
  - 배송 주기
  - 다음 주문 예정일

[주요 액션]
- 각 항목: 상세 보기
```

Proposed UX Design:

- 목록 항목은 하나의 SKU만 대상으로 표시한다.
- 구독 상태, 변경, 건너뛰기, 일시정지, 재개, 해지 액션은 표시하지 않는다.
- 목록이 비어 있으면 빈 상태를 표시한다.

### UX-SCREEN-006 내 구독 상세

```text
[상단]
- 이전으로
- 내 구독 상세

[본문]
- 구독 ID
- 상품명
- SKU 용량 또는 구성
- SKU 표시 가격
- 수량
- 배송 주기
- 구독 생성일
- 다음 주문 예정일

[주요 액션]
- 내 구독 목록으로
- 상품 상세 보기
```

Proposed UX Design:

- 구독 상태, 결제, 재고, 배송, 변경 액션은 표시하지 않는다.
- 다음 배송 예정일은 표시하지 않는다.

### UX-SCREEN-007 로그인 리디렉션 동작

```text
[동작]
- 비회원이 구독 생성, 내 구독 목록 또는 내 구독 상세에 접근
- 별도 중간 페이지를 표시하지 않음
- 로그인 화면으로 이동
- 로그인 화면에서 필요하면 로그인 필요 이유를 안내
```

Proposed UX Design:

- 이 문서는 로그인 화면 자체와 인증 구현 방식을 설계하지 않는다.
- 비회원 제한 접근은 중간 안내 화면을 표시하지 않고 로그인 화면으로 이동하는 동작으로 정의한다.
- 로그인 후 원래 화면으로 돌아오는지는 승인된 요구사항에 없으므로 확정하지 않는다.

### UX-SCREEN-008 조회할 수 없는 상품·SKU·구독 상태

```text
[상단]
- 확인할 수 없음

[본문]
- 상품: 상품을 확인할 수 없습니다.
- SKU: 선택한 옵션을 구독할 수 없습니다.
- 구독: 구독을 확인할 수 없습니다.

[주요 액션]
- 일시적 상품 목록·상세 조회 실패: 다시 시도
- 존재하지 않는 상품: 상품 목록으로 이동
- 존재하지 않거나 구독 불가능한 SKU: 상품 상세로 돌아가 다른 옵션 선택, 상품 목록으로 이동
- 존재하지 않거나 다른 회원 소유의 구독: 내 구독 목록으로 이동
```

Proposed UX Design:

- 존재하지 않는 상품 또는 SKU는 구독 대상으로 선택할 수 없음을 안내한다.
- 구독 불가능 SKU는 선택 단계에서 비활성화하고, 직접 접근 또는 제출 시도에서는 오류 상태로 안내한다.
- 일시적인 목록·상세 조회 실패에만 재시도 액션을 제공한다.
- 존재하지 않는 상품은 상품 목록으로 이동하는 액션을 우선 제공한다.
- 존재하지 않거나 구독 불가능한 SKU는 상품 상세에서 다른 옵션을 선택하거나 상품 목록으로 이동하게 한다.
- 존재하지 않거나 다른 회원 소유의 구독은 같은 “구독을 확인할 수 없습니다.” 상태로 처리하고, 소유자나 존재 여부를 노출하지 않는다.
- 조회할 수 없는 구독 상태에서는 내 구독 목록으로 이동하는 액션을 우선 제공하고, 재시도를 기본 액션으로 두지 않는다.
- HTTP 상태, 오류 코드, 오류 JSON은 확정하지 않는다.

## 접근성 기준

- 키보드만으로 상품 카드 상세 이동, SKU 선택, 수량 입력, 배송 주기 선택, 제출, 재시도가 가능해야 한다.
- SKU 선택은 라디오 그룹 또는 동등한 단일 선택 패턴으로 구현하고 각 항목에 명확한 이름을 제공한다.
- 수량 입력에는 `수량` label과 1~10 허용 범위 설명을 연결한다.
- 배송 주기 선택에는 `배송 주기` group label을 제공한다.
- 오류 메시지는 해당 입력과 programmatically 연결한다.
- `구독 만들기` 버튼은 누락 입력이 있어도 실행 가능해야 하며, 실행 시 생성 요청 전 입력 오류를 표시한다.
- 제출 오류가 여러 개이면 오류 요약을 제공하고, 포커스는 오류 요약 또는 첫 오류 입력으로 이동한다.
- 비활성 상태는 색상만으로 표현하지 않고 텍스트 또는 아이콘과 함께 설명한다.
- 모든 주요 인터랙션 요소에는 명확한 포커스 표시가 있어야 한다.
- 로딩, 성공, 오류 상태는 보조기기가 인지할 수 있게 상태 메시지를 알린다.
- 날짜는 `2026. 7. 31.`처럼 읽을 수 있는 텍스트로 제공하고, 다음 배송 예정일과 혼동되지 않게 “다음 주문 예정일” label을 붙인다.
- 가격은 SKU 표시 가격임을 label로 구분하고 총 결제 금액처럼 표현하지 않는다.
- 모바일 터치 대상은 최소 44px에 준하는 터치 영역을 확보한다.
- 읽기 순서는 상품 정보, SKU 선택, 수량, 배송 주기, 검토, 제출 순서를 유지한다.

## QA 검증 메모

- 상품 목록에 상품 ID, 상품명, 대상 동물, 설명, SKU 가격 요약, 구독 가능 SKU 존재 여부가 표시되는지 확인한다.
- 상품 상세에서 구독 불가능 SKU가 선택 불가로 표현되는지 확인한다.
- 수량 1, 10은 허용하고 0, 11은 오류로 표현되는지 확인한다.
- 배송 주기는 2주, 4주, 8주만 선택 가능한지 확인한다.
- SKU, 수량, 배송 주기 누락 상태에서도 `구독 만들기` 실행이 가능한지 확인한다.
- 유효하지 않은 입력에서는 구독 생성 요청으로 진행하지 않는지 확인한다.
- 입력 오류가 필드 아래와 오류 요약으로 표시되는지 확인한다.
- 제출 오류 후 포커스가 오류 요약 또는 첫 번째 유효하지 않은 입력으로 이동하는지 확인한다.
- 제출 중에는 버튼과 SKU·수량·배송 주기 입력이 잠기고 중복 제출이 방지되는지 확인한다.
- 비회원 제한 화면 접근 시 별도 중간 화면 없이 로그인 화면으로 이동하는지 확인한다.
- 로그인 후 원래 화면 복귀는 확정 동작으로 테스트하지 않는다.
- 다른 회원 소유 구독과 존재하지 않는 구독이 같은 “구독을 확인할 수 없습니다.” 문구로 표현되는지 확인한다.
- 구독 소유자, 권한, 존재 여부의 세부 정보가 노출되지 않는지 확인한다.
- 생성 성공 후 구독 ID와 확정된 다음 주문 예정일을 확인할 수 있는지 확인한다.
- 다음 주문 예정일이 `Asia/Seoul` 기준 날짜 단위로 표시되는지 확인한다.
- 다음 주문 예정일에 휴일·주말·영업일 보정이 적용된 것처럼 표현되지 않는지 확인한다.
- 내 구독 목록과 상세에 구독 상태, 배송 예정일, 변경 액션이 표시되지 않는지 확인한다.
- 다음 배송 예정일이 어떤 화면에도 표시되지 않는지 확인한다.
