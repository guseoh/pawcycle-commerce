# PS-003 PO → UX 인수인계

## 전달 목적

UX-001에서 열린 상태였던 네 가지 제품 정책이 PS-003에서 Approved Product Decision으로 승인됐음을 UX-002에 전달한다.

UX-002는 제품 정책을 다시 변경하지 않고, UX-001 문서와 화면 흐름을 PS-003 승인 결정에 맞게 동기화한다.

## 승인 근거

- `docs/product/PS-003-ux-product-decisions.md`
- `docs/product/PS-002-first-mvp-requirements.md`
- `docs/domain/DOMAIN-001-first-mvp-subscription-domain.md`
- `docs/design/UX-001-first-mvp-subscription-experience.md`

## 승인된 Product Decision

1. 비회원 상품 목록·상세 접근 허용
2. 로그인 후 안전한 서비스 내부 GET 경로 복귀
3. 구독 생성 성공 후 생성된 구독 상세 화면으로 이동
4. 생성 전 정확한 다음 주문 예정일 미표시

UX-001의 관련 열린 질문은 PS-003 이후 더 이상 Open Decision으로 유지하지 않는다.

## UX-002에서 반영할 상태

- 비회원은 상품 목록을 조회할 수 있다.
- 비회원은 상품 상세, SKU와 구독 가능 여부를 확인할 수 있다.
- 구독 생성, 내 구독 목록, 내 구독 상세는 로그인한 회원만 사용할 수 있다.
- 비회원이 보호 기능에 접근해 로그인 화면으로 이동한 경우 로그인 성공 후 안전한 서비스 내부 GET 경로로 복귀한다.
- 유효한 내부 복귀 경로가 없으면 상품 목록으로 이동한다.
- 외부 URL과 검증되지 않은 임의 경로로 이동하지 않는다.
- 로그인 후 입력 폼 상태를 자동 복원하지 않는다.
- 이전 POST 요청을 자동으로 다시 실행하지 않는다.
- 구독 생성 성공 후 생성된 구독 상세 화면으로 이동한다.
- 구독 상세 화면에서 생성 성공 안내, 구독 ID와 다음 주문 예정일을 표시한다.
- 별도의 영구적인 구독 생성 성공 페이지는 만들지 않는다.
- 구독 생성 전에는 정확한 다음 주문 예정일 날짜를 표시하지 않는다.
- 입력 검토 단계에는 선택한 배송 주기와 “다음 주문 예정일은 구독을 만든 뒤 확인할 수 있습니다.” 안내만 표시한다.
- 생성 성공 후, 내 구독 목록과 내 구독 상세에서는 서버가 확정한 다음 주문 예정일을 표시한다.

## UX-002 권장 변경

- UX-001 열린 질문 표의 네 항목을 `Resolved`로 변경한다.
- PS-003 문서를 승인 근거로 연결한다.
- `Proposed UX Design` 중 이번 결정에 해당하는 항목을 승인 입력으로 승격한다.
- 구독 생성 성공 화면을 별도 영구 화면이 아닌 구독 상세 이동으로 정리한다.
- 로그인 복귀 시 폼 상태와 POST 요청을 복원하지 않음을 명시한다.
- 실제 프론트엔드 구현은 수행하지 않는다.

## UX에서 확정하지 말아야 할 항목

- API 요청·응답 JSON
- HTTP 상태와 오류 코드
- URL과 라우트 문자열
- Spring Security 구현 방식
- 로그인 복귀 경로 저장 구현 방식
- Open Redirect 방지 코드
- 프론트엔드 라우팅 구현

## 후속 역할 영향

- UX-002: 승인 결정 반영과 화면 흐름 동기화
- ARCH-001: 로그인 복귀 경로와 보안 경계 검토
- API-001: 공개 조회, 인증 실패, 구독 생성 성공 응답 계약 정리
- 인증 설계: 안전한 내부 GET 경로 복귀와 fallback 정책 구현 방식 검토
- QA: 공개 탐색, 보호 기능 접근, 로그인 후 복귀, 생성 후 상세 이동, 생성 전 예정일 미표시 검증

## 제한

- 이 인수인계는 제품 정책만 전달한다.
- API 계약, 인증 구현, 라우팅 구현은 확정하지 않는다.
- UX-002에서 제품 정책을 다시 변경하려면 Product Owner 승인이 필요하다.
