# PS-002 PO → BE 인수인계

## 전달 목적

PS-002에서 구체화한 첫 번째 수직 MVP 기능 요구사항과 인수 조건을 DOMAIN-001 및 Backend Engineer 역할이 추측 없이 이어받도록 전달한다.

DOMAIN-001의 역할은 Backend Engineer이며 작업 브랜치는 `feat/be`다.

이 문서는 도메인 상세 설계 산출물이 아니다. JPA Entity, JPA 연관관계, 테이블, API DTO와 구현 코드는 이 문서에서 확정하지 않는다.

- PR `#4`: `docs(product): 1차 MVP 도메인과 흐름 정리`
- `docs/product/PS-001-first-mvp-domain-scope.md`
- `docs/product/PS-002-first-mvp-requirements.md`
- `docs/domain/glossary.md`
- `docs/domain/rules.md`

## DOMAIN-001에 전달할 확정 입력

### 제품 범위

- PS-001은 구독 생성·조회 도메인을 검증하는 첫 번째 수직 MVP다.
- 첫 번째 MVP의 상품 예시와 검증 범위는 개와 고양이용 사료다.
- 상품과 SKU 도메인 개념은 사료 전용으로 제한하지 않는다.
- 일반 구매, 장바구니, 결제, 재고, 배송, 정기 주문 자동 생성은 제외한다.

### Member

- 첫 번째 MVP의 사용자는 로그인한 회원이다.
- 회원은 상품을 탐색한다.
- 회원은 구독 가능한 SKU 하나를 대상으로 구독을 생성한다.
- 회원은 자신의 구독 목록과 상세를 조회한다.
- 회원은 다른 회원의 구독을 조회할 수 없다.

### Product

- 상품은 회원에게 하나의 반려동물 소모품으로 인식되는 공통 상품 정보다.
- 상품 목록 표시 정보는 상품 ID, 상품명, 대상 동물, 짧은 설명, SKU별 표시 가격, 구독 가능한 SKU 존재 여부다.
- 상품 상세 표시 정보는 상품명, 상품 설명, 대상 동물, SKU 목록, SKU별 용량 또는 구성, SKU별 표시 가격, 구독 가능 여부, 선택 가능한 배송 주기다.

### SKU

- SKU는 용량 또는 구성 등 선택 항목이 확정된 구독 선택 단위다.
- 회원은 존재하며 구독 가능한 SKU만 구독할 수 있다.
- 재고, 품절과 일반 판매 상태는 이번 구독 생성 성공·실패 조건에 연결하지 않는다.
- 구독 가능 여부의 도메인 표현은 DOMAIN-001에서 구체화한다.

### Subscription

- 첫 번째 MVP에서 구독 하나는 SKU 하나만 대상으로 한다.
- 복수 SKU 구독은 Post-MVP다.
- 구독은 회원, SKU 하나, 수량, 배송 주기, 구독 생성일, 다음 주문 예정일을 설명해야 한다.
- 첫 번째 MVP에서는 구독 상태 모델을 사용하지 않는다.
- 회원은 자신의 구독만 조회할 수 있다.

## 구독 생성 요구사항

### 성공 조건

- 로그인한 회원
- 존재하는 SKU
- 구독 가능한 SKU
- 수량 1~10
- 배송 주기 2주, 4주, 8주 중 하나

### 성공 결과

- 구독이 생성된다.
- 생성된 구독 상세를 확인할 수 있다.
- 다음 주문 예정일을 확인할 수 있다.

### 실패 조건

- 비로그인
- 존재하지 않는 SKU
- 구독 불가능한 SKU
- 수량 범위 위반
- 허용되지 않은 배송 주기
- 필수 입력 누락

HTTP 상태, 오류 코드와 오류 응답 JSON은 API-001에서 확정한다.

## 조회 요구사항

### 내 구독 목록 표시 정보

- 구독 ID
- 상품명
- SKU
- 수량
- 배송 주기
- 다음 주문 예정일

### 내 구독 상세 표시 정보

- 구독 ID
- 상품명
- SKU
- SKU 표시 가격
- 수량
- 배송 주기
- 구독 생성일
- 다음 주문 예정일

### 접근 규칙

- 비회원이 구독 생성·조회 화면에 접근하면 로그인 화면으로 이동한다.
- 회원은 다른 회원의 구독을 조회할 수 없다.
- 다른 회원의 구독 접근은 조회할 수 없는 구독으로 처리한다.

## 날짜와 배송 주기

- 배송 주기는 2주, 4주, 8주다.
- 사용자는 최초 주문 예정일을 선택하지 않는다.
- 다음 주문 예정일은 구독 생성일에 선택한 배송 주기를 더해 계산한다.
- 날짜 단위로 관리한다.
- 기준 시간대는 Asia/Seoul이다.
- 휴일, 주말과 영업일 보정은 하지 않는다.
- 다음 주문 예정일만 제공하며 배송 예정일은 제공하지 않는다.
- 실제 정기 주문 자동 생성은 포함하지 않는다.
- 계약에서 사용할 날짜 기준 형식 후보는 `YYYY-MM-DD`다.
- 사용자 화면 표시는 `YYYY. M. D.`다.
- API 계약의 최종 날짜 표현은 API-001에서 확정한다.

- 회원과 구독의 관계
- 첫 번째 MVP 단일 SKU 제한

- 다음 주문 예정일 계산 책임
- Asia/Seoul과 날짜 단위 처리
- 구독 생성 정상·실패 시나리오

## 주의 사항

- 전역 도메인 용어집은 구독을 하나 이상의 상품을 반복 배송받는 상위 개념으로 정의한다.
- PS-002는 첫 번째 MVP에서만 구독 하나당 SKU 하나로 제한한다.
- 전역 구독 용어와 첫 번째 MVP 단일 SKU 제한의 정합성은 DOMAIN-001에서 용어 수준으로 정리한다.
- 구독 상태 모델은 첫 번째 MVP에서 사용하지 않는다.
- 재고, 품절과 일반 판매 상태는 이번 구독 생성 성공·실패 조건에 연결하지 않는다.
