# UX-001 UX → FE 인수인계

## 전달 목적

UX-001에서 정의한 1차 MVP 구독 사용자 흐름과 화면 상태를 프론트엔드 엔지니어가 추측 없이 구현할 수 있도록 전달한다.

## 기준 문서

- `docs/design/UX-001-first-mvp-subscription-experience.md`
- `docs/product/PS-002-first-mvp-requirements.md`
- `docs/domain/DOMAIN-001-first-mvp-subscription-domain.md`

## 구현 대상 화면

- 상품 목록
- 상품 상세 및 구독 입력
- 구독 생성 처리 상태
- 구독 생성 성공 결과 또는 구독 상세
- 내 구독 목록
- 내 구독 상세
- 로그인 이동 상태
- 조회할 수 없는 상품·SKU·구독 상태

## 핵심 구현 흐름

```text
상품 목록
→ 상품 상세
→ SKU 선택
→ 수량 입력
→ 배송 주기 선택
→ 입력 검토
→ 구독 생성
→ 생성된 구독 상세 확인
```

```text
내 구독 목록
→ 내 구독 상세
```

## 필수 UI 규칙

- SKU는 하나만 선택할 수 있다.
- 구독 불가능 SKU는 선택 불가로 표현한다.
- 수량은 1~10만 허용한다.
- 배송 주기는 2주, 4주, 8주만 표시한다.
- 다음 주문 예정일은 `YYYY. M. D.` 형식으로 표시한다.
- 다음 배송 예정일은 표시하지 않는다.
- 구독 상태, 결제, 재고, 배송, 구독 변경 액션은 표시하지 않는다.
- 구독 생성 중에는 중복 제출을 방지한다.

## 입력 오류 문구

| 상황 | 문구 |
| --- | --- |
| SKU 미선택 | 구독할 옵션을 선택해 주세요. |
| 구독 불가능 SKU | 이 옵션은 현재 구독 대상으로 사용할 수 없습니다. |
| 수량 누락 | 수량을 입력해 주세요. |
| 수량 1 미만 | 수량은 1개 이상이어야 합니다. |
| 수량 10 초과 | 수량은 최대 10개까지 선택할 수 있습니다. |
| 배송 주기 미선택 | 배송 주기를 선택해 주세요. |
| 허용되지 않은 배송 주기 | 배송 주기는 2주, 4주, 8주 중에서 선택해 주세요. |
| 중복 제출 방지 | 구독을 만들고 있습니다. |

## 접근과 제한 상태

- 비회원이 구독 생성, 내 구독 목록, 내 구독 상세에 접근하면 로그인 화면으로 이동한다.
- 로그인 후 원래 화면으로 복귀할지 여부는 아직 확정하지 않는다.
- 다른 회원의 구독 또는 존재하지 않는 구독은 “구독을 확인할 수 없습니다.” 상태로 표현한다.
- HTTP 상태와 오류 응답 JSON은 API-001에서 확정한다.

## 접근성 구현 주의

- SKU 선택은 키보드로 조작 가능한 단일 선택 패턴으로 구현한다.
- 수량 입력과 배송 주기 선택에는 명확한 label을 연결한다.
- 오류 메시지는 해당 필드와 연결한다.
- 제출 오류가 여러 개이면 오류 요약을 제공하거나 첫 오류로 포커스를 이동한다.
- 로딩, 성공, 오류 상태는 보조기기에 알린다.
- 비활성 상태는 색상만으로 표현하지 않는다.
- 모바일 주요 액션은 충분한 터치 영역을 확보한다.

## 반응형 구현 주의

- 모바일은 단일 컬럼을 기본으로 한다.
- 데스크톱은 상품 정보와 구독 입력을 2컬럼으로 나눌 수 있다.
- 긴 상품명과 SKU 텍스트는 줄바꿈을 허용한다.
- 주요 액션 버튼은 모바일에서 본문을 가리지 않게 배치한다.

## 열린 질문

- 로그인 후 원래 화면으로 복귀할지 여부
- 비회원 상품 목록·상세 접근 허용 여부
- 구독 생성 성공 후 상세 라우트로 즉시 이동할지, 결과 화면을 먼저 보여줄지 여부
- 다음 주문 예정일 예상값의 클라이언트 사전 계산 여부

## FE에서 확정하지 말아야 할 항목

- API HTTP 상태
- 오류 코드
- 오류 응답 JSON
- 인증 구현 방식
- 상품 검색·정렬·필터·페이지네이션
- 결제, 재고, 배송, 구독 변경 기능
