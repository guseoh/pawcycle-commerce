# UX-001 UX → TL 인수인계

## 전달 목적

UX-001 PR #11 검토 결과에 따라 1차 MVP 구독 사용자 흐름의 열린 결정을 Tech Lead와 Product Owner가 검토할 수 있도록 전달한다.

Product Decision, 인증 흐름, API 계약이 확정되기 전에는
프론트엔드가 열린 질문을 임의로 결정하거나 구현하지 않는다.

## 승인된 입력

- `docs/product/PS-002-first-mvp-requirements.md`
- `docs/domain/DOMAIN-001-first-mvp-subscription-domain.md`
- 상품 검증 범위는 개와 고양이용 사료다.
- 구독 하나는 SKU 하나만 대상으로 한다.
- 수량은 1~10이다.
- 배송 주기는 2주, 4주, 8주다.
- 구독 생성과 내 구독 조회는 로그인한 회원만 사용할 수 있다.
- 비회원이 구독 생성, 내 구독 목록, 내 구독 상세에 접근하면 로그인 화면으로 이동한다.
- 다른 회원의 구독 상세 접근은 조회할 수 없는 구독으로 처리한다.
- HTTP 상태, 오류 코드, API 응답 구조, 인증 구현 방식은 확정하지 않는다.

## UX 설계 요약

- 상품 상세 화면 안에서 SKU, 수량, 배송 주기를 선택하고 구독을 생성하는 흐름을 Proposed UX Design으로 정리했다.
- `구독 만들기` 버튼은 기본 활성 상태이며, 입력 오류가 있으면 생성 요청을 보내지 않고 필드 오류와 오류 요약을 표시한다.
- 제출 중에는 버튼과 SKU, 수량, 배송 주기 입력을 잠그고 중복 제출을 방지한다.
- 비회원 제한 접근은 별도 중간 화면 없이 로그인 화면으로 이동하는 동작으로 정의했다.
- 존재하지 않는 구독과 다른 회원 소유 구독은 모두 `구독을 확인할 수 없습니다.`로 표현한다.
- 입력 단계에서는 다음 주문 예정일의 구체 날짜를 계산하지 않고, 생성 성공 결과와 구독 상세에서 확정된 값을 표시한다.

## 권장 UX 기본안

- 상품 목록과 상품 상세는 비회원에게도 탐색 가능하게 두는 안을 권장한다.
- 로그인 후 안전한 내부 경로라면 원래 요청 화면으로 복귀하는 안을 권장한다.
- 구독 생성 성공 후 생성된 구독 상세로 즉시 이동하는 안을 권장한다.
- 생성 전 입력 검토에서는 다음 주문 예정일 날짜를 미리 계산하지 않고, 선택한 배송 주기와 생성 후 확인 안내만 보여주는 안을 권장한다.

## 후속 작업에 미치는 영향

- API-001은 로그인 실패, 조회할 수 없는 구독, 생성 성공 응답에서 어떤 값을 제공할지 확정해야 한다.
- Security 또는 인증 설계는 제한 화면 접근 후 로그인 화면 이동과 원래 경로 복귀 여부를 결정해야 한다.
- FE는 열린 결정이 확정되기 전까지 라우팅, 인증 가드, 성공 이동, 사전 날짜 계산을 구현 범위로 확정하지 않는다.
- QA는 열린 결정이 확정되기 전까지 해당 항목을 확정 동작으로 테스트하지 않는다.

## 불변 조건

- 이 문서는 사용자 승인 전까지 Proposed UX Design 상태를 유지한다.
- PS-002와 DOMAIN-001의 승인 입력을 넘는 신규 제품 정책을 만들지 않는다.
- API HTTP 상태, 오류 코드, 오류 응답 JSON, 인증 구현 방식은 확정하지 않는다.
- 다음 배송 예정일은 어떤 화면에도 표시하지 않는다.
- 구독 상태, 결제, 재고, 배송, 구독 변경·해지·일시정지 기능은 포함하지 않는다.

## 검증 결과

- `docs/design/UX-001-first-mvp-subscription-experience.md`에 검토 보완 사항을 반영했다.
- `docs/reports/UX-001/ux-report.md`에 보완 결과와 남은 Product Decision을 기록했다.
- 기존 FE 인수인계 문서는 삭제하고, 이 TL 인수인계 문서로 교체했다.
- 최종 명령 검증 결과는 UX-001 작업 보고서와 PR #11 본문에 함께 기록한다.

## 중단 조건

- 비회원 상품 탐색 정책을 확정해야만 구현할 수 있는 경우 중단한다.
- 로그인 후 원래 화면 복귀 여부를 확정해야만 인증 흐름을 구현할 수 있는 경우 중단한다.
- 구독 생성 성공 후 이동 방식이 API 응답 구조를 요구하는 경우 API-001 전까지 중단한다.
- 생성 전 다음 주문 예정일 사전 계산을 구현하려면 날짜 계산 책임과 API 계약 확정 전까지 중단한다.
- 열린 질문을 프론트엔드 또는 QA가 임의로 확정해야 하는 상황이면 중단하고 TL 또는 Product Owner 결정을 요청한다.
