## 사용 가능한 결과

## 인수 조건과 추적성

- 경량은 보고서·인수인계 없이 통과한다.
- 일반은 실질 보고서와 조건부 인수인계를 사용한다.
- 고위험은 승인·적용 전후·독립 검증·복구 및 롤백 증거를 유지한다.

## 확정된 결정

- 등급이 불명확하면 일반으로 시작한다.
- 외부 계약·데이터·보안·운영·복구 위험을 발견하면 고위험으로 승격한다.
- prompt 길이 예산은 CI 실패 조건이 아니다.

## 미확정 결정

## 승인 필요 항목

- 새 작업에서 등급 하향 또는 고위험 범위 확대가 필요하면 사용자 또는 Tech Lead 확인이 필요하다.

## 다음 역할의 입력

- 작업 ID, 작업 등급, 현재 작업의 delta, 포함·제외 범위, 완료·검증 조건과 중단 조건만 prompt에 적는다.

## 지켜야 할 규칙

- 기존 보고서와 인수인계를 소급 수정하지 않는다.
- 공통 지침·역할 원문·Git 절차·template을 prompt에 반복하지 않는다.
- 실제 다음 소비자가 없으면 인수인계를 만들지 않는다.

## 적용·실행 방법

- PR과 일반·고위험 보고서의 `작업 등급` 필드를 채운다.
- 로컬 검증은 `python scripts/validate-task-artifacts.py --task-id <TASK-ID> --task-grade <등급>` 형식을 사용한다.
