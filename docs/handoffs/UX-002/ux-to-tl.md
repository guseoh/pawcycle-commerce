# UX-002 UX → TL 인수인계

## 전달 목적

UX-002에서 PS-003 승인 Product Decision 네 가지를 UX-001 사용자 경험 설계 문서에 반영했음을 TL에게 전달한다.

후속 ARCH-001, API-001, 인증 설계, FE, QA가 사용할 승인 UX 입력과 아직 기술 결정으로 남은 항목을 구분한다.

## 승인된 제품 결정 반영 결과

| PS-003 결정 | UX 문서 반영 |
| --- | --- |
| 비회원 상품 목록·상세 접근 허용 | 상품 목록, 상품 상세, SKU와 구독 가능 여부를 비회원과 회원 모두 확인 가능한 공개 기능으로 정리했다. |
| 로그인 후 안전한 내부 GET 복귀 | 보호 기능 접근 시 로그인 화면으로 이동하고, 로그인 성공 후 안전한 내부 GET 경로로 복귀하며, 유효한 복귀 경로가 없으면 상품 목록으로 이동하도록 정리했다. |
| 구독 생성 성공 후 생성된 구독 상세 이동 | 별도의 영구 성공 페이지 없이 생성된 구독 상세 화면으로 이동하고, 상세에서 생성 성공 안내, 구독 ID와 다음 주문 예정일을 보여주도록 정리했다. |
| 생성 전 정확한 다음 주문 예정일 미표시 | 상품 상세 입력 검토에서는 정확한 날짜를 표시하지 않고, 선택한 배송 주기와 생성 후 확인 안내만 보여주도록 정리했다. |

## 후속 역할이 사용할 UX 입력

- 상품 목록과 상품 상세는 공개 탐색 범위다.
- 구독 생성, 내 구독 목록, 내 구독 상세는 로그인한 회원만 사용할 수 있다.
- 비회원이 `구독 만들기`, 내 구독 목록, 내 구독 상세에 접근하면 로그인 화면으로 이동한다.
- 로그인 성공 후 안전한 서비스 내부 GET 복귀 경로가 있으면 해당 경로로 복귀한다.
- 유효한 내부 복귀 경로가 없으면 상품 목록으로 이동한다.
- 외부 URL과 검증되지 않은 임의 경로로 이동하지 않는다.
- 로그인 후 입력 폼 상태를 자동 복원하지 않는다.
- 이전 POST 요청을 자동으로 다시 실행하지 않는다.
- 구독 생성 성공 후 생성된 구독 상세 화면으로 이동한다.
- 구독 상세 화면에는 생성 성공 안내, 구독 ID와 서버가 확정한 다음 주문 예정일을 표시한다.
- 별도의 영구적인 구독 생성 성공 페이지는 만들지 않는다.
- 구독 생성 전 입력 검토에는 “다음 주문 예정일은 구독을 만든 뒤 확인할 수 있습니다.” 안내만 표시한다.
- 구독 생성 성공 후, 내 구독 목록, 내 구독 상세에서는 서버가 확정한 다음 주문 예정일을 `YYYY. M. D.` 형식으로 표시한다.
- 다음 주문 예정일은 `Asia/Seoul` 기준 날짜 단위이며 휴일·주말·영업일 보정을 표현하지 않는다.
- 다음 배송 예정일은 표시하지 않는다.

## Deferred Technical Decision

- 구체 URI와 라우트 문자열
- 로그인 복귀 경로 저장 방식
- Open Redirect 방지 구현
- 인증 실패 HTTP 상태와 오류 응답
- 구독 생성 성공 응답 JSON
- Spring Security 구현 방식
- 프론트엔드 라우팅 구현 방식

## 후속 작업 영향

- ARCH-001은 로그인 복귀 경로 저장, 내부 GET 경로 검증, Open Redirect 방지 책임 위치를 검토해야 한다.
- API-001은 공개 상품 조회, 보호 기능 인증 실패, 구독 생성 성공 응답의 식별자 제공과 오류 표현을 계약으로 정리해야 한다.
- 인증 설계는 안전한 내부 복귀 경로와 상품 목록 fallback, 외부 URL 차단 방식을 설계해야 한다.
- FE는 UX 문서의 개념 화면과 상태를 구현하되 구체 라우트 문자열과 인증 저장 방식은 승인된 기술 설계를 따라야 한다.
- QA는 공개 상품 탐색, 보호 기능 로그인 이동, 로그인 후 복귀, POST 자동 재실행 금지, 생성 후 상세 이동, 생성 전 예정일 미표시를 검증해야 한다.

## 검증 결과

- UX-002 작업 산출물 검증을 통과했다.
- 커밋 메시지 규칙 검증을 통과했다.
- PS-003 승인 근거와 네 가지 `Resolved` 상태를 확인했다.
- 금지 경로인 `docs/product/**`, `docs/domain/**`, `backend/**`, `frontend/**`, `infra/**`를 변경하지 않았음을 확인했다.
- 자세한 검증 명령과 결과는 `docs/reports/UX-002/ux-report.md`를 따른다.

## 중단 조건

- PS-003 결정과 UX-001 문서가 충돌해 Product Owner 재결정이 필요한 경우
- UX 문서 변경만으로 해결할 수 없고 제품 요구사항 변경이 필요한 경우
- API·인증·라우팅 구현 방식을 확정해야만 후속 작업이 가능한 경우
- 신규 기능 또는 의존성이 필요한 경우
- Secret 또는 민감정보 노출이 의심되는 경우
