# PS-001 PO to BE 인수인계

## 전달 목적

DOMAIN-001 및 Backend Engineer 역할이 제품 비전과 MVP 범위를 바탕으로 업무 규칙과 도메인 경계를 검토할 수 있도록 제품 입력을 전달한다.

이 문서는 제품 정책 인수인계이며 도메인 설계 산출물이 아니다. 엔티티, 테이블, API, 상태 전이, 불변식은 이 문서에서 확정하지 않는다.

## 완료된 작업

- PawCycle Commerce의 서비스 비전과 사용자 문제를 정리했다.
- 1차 사용자와 2차 사용자를 정의했다.
- MVP 포함 범위, 제외 범위, Post-MVP 후보를 구분했다.
- 단건 구매, 정기 구독, 가격, 품절, 결제 실패, 주문 취소, 배송, 운영자 기능에 대한 PS-001 기준 제품 결정을 기록했다.
- 이전 `spec/PS-001-service-vision-mvp` 브랜치를 비교하고 제품 판단만 수동 반영했다.

## DOMAIN-001 입력

다음 항목은 제품 정책으로 확정한 입력이다. DOMAIN-001에서는 이를 바탕으로 업무 규칙을 검토하되, 제품 문서의 문장을 그대로 구현 설계로 간주하지 않는다.

1. 대상은 반려견/반려묘 소모품으로 제한한다.
2. 단건 구매와 정기 구독을 모두 MVP에 포함한다.
3. 구매와 구독 선택 단위는 SKU다.
4. 하나의 구독은 하나의 SKU만 대상으로 한다.
5. 구독 주기는 2주, 4주, 8주다.
6. 첫 배송일 자유 선택은 MVP에서 제외한다.
7. 구독 관리는 수량 변경, 다음 회차 건너뛰기, 일시정지, 재개, 해지를 포함한다.
8. 구독 변경은 다음 회차 생성 예정일 전날 23:59 KST까지 허용한다.
9. 해지한 구독은 재개할 수 없고 새 구독을 만들어야 한다.
10. 정기 회차 가격은 회차 생성 시점의 현재 가격을 기준으로 한다.
11. 품절이면 해당 회차 진행을 보류하고 사용자에게 화면에서 알린다.
12. 판매 중단 SKU는 신규 구매와 신규 구독을 막고 기존 이력 확인은 허용한다.
13. 결제 실패 시 자동 재시도는 하지 않고 사용자가 수동 재시도한다.
14. 결제 실패가 있어도 구독 자체는 유지한다.
15. 주문 취소는 배송 준비 전까지만 사용자에게 허용한다.
16. 단건 주문과 정기 주문에는 같은 단일 기본 배송 정책을 적용한다.
17. 운영자는 상품 판매 가능 여부, 재고, 주문, 구독 현황을 최소한으로 확인해야 한다.

## DOMAIN-001에서 검토할 질문

- 제품 정책의 용어를 어떤 도메인 용어로 정리할지
- 다음 회차, 회차 생성, 배송 준비 전, 품절 보류 같은 표현을 어떤 업무 규칙으로 설명할지
- 구독 변경 마감 시점이 사용자의 행동과 운영자의 처리에 어떤 영향을 주는지
- 품절, 판매 중단, 결제 실패, 해지 후 재개 불가를 어떤 규칙으로 다룰지
- 주문 취소가 허용되는 경계가 제품 경험과 충돌하지 않는지

## 지켜야 할 규칙

- PS-001 문서를 근거로 도메인을 재선정하지 않는다.
- 본 인수인계는 구현 방식, 저장 방식, API 계약을 지시하지 않는다.
- 추천안이 PR 리뷰에서 조정될 수 있음을 유지한다.
- 관리자 기능은 최소 운영 필요성만 전달하며 화면 형태는 후속 UX/FE 작업에서 결정한다.
