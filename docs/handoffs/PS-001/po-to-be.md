# PS-001 PO → BE 인수인계

## 전달 목적

첫 번째 수직 MVP의 제품 도메인 범위와 대표 사용자 흐름을 DOMAIN-001 및 Backend Engineer 역할이 추측 없이 이어받도록 전달한다.

이 문서는 도메인 상세 설계 산출물이 아니다. 엔티티, 테이블, API, 클래스 이름, 상태 전이, 불변식은 이 문서에서 확정하지 않는다.

## 확정된 1차 MVP 범위

PS-001은 구독 생성과 조회에 필요한 최소 제품 흐름과 핵심 도메인 규칙을 검증하는 첫 번째 수직 MVP다.

로그인한 회원이 개와 고양이용 사료 상품을 탐색하고, 구독 가능한 SKU 하나를 선택한 뒤, 수량과 배송 주기를 지정해 정기배송 구독을 생성하고 자신의 구독과 다음 주문 예정일을 확인한다.

일반 구매는 상위 제품 방향으로 유지하지만 첫 번째 수직 MVP 구현 범위에서는 제외한다.

## 포함 도메인

- 회원
- 상품
- 상품 옵션 또는 SKU
- 구독
- 배송 주기
- 다음 주문 예정일

배송 주기와 다음 주문 예정일은 현재 단계에서 별도 엔티티가 아니라 구독을 설명하는 제품 개념 또는 값이다.

## 도메인별 제품 의미

### 회원

- 첫 번째 MVP의 사용자는 로그인한 회원이다.
- 회원은 상품을 탐색한다.
- 회원은 특정 SKU를 대상으로 구독을 생성한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.
- 비회원 구독과 다른 회원의 구독 조회는 제외한다.

### 상품

상품은 사용자에게 하나의 반려동물 소모품으로 인식되는 공통 상품 정보다. 첫 번째 수직 MVP의 예시와 검증 데이터는 개와 고양이용 사료를 중심으로 한다.

상품과 SKU라는 도메인 개념은 사료에 종속시키지 않는다. 간식, 고양이 모래, 배변 패드, 위생용품은 후속 상품 범위다.

### 상품 옵션 또는 SKU

상품 옵션 또는 SKU는 용량, 구성, 맛 등 선택 항목이 확정된 실제 선택 단위다.

첫 번째 MVP에서 회원은 상품 자체가 아니라 구독 가능한 SKU를 구독 대상으로 선택한다. 판매 상태와 구독 가능 여부는 다른 제품 개념이다.

- 판매 상태: 일반적인 상품 판매 가능 여부
- 구독 가능 여부: 해당 SKU를 정기배송 구독 대상으로 선택할 수 있는지

회원은 구독 가능한 SKU만 구독할 수 있다.

상태 enum 이름, 상태 코드, DB 컬럼, 판매 상태 전이, 구독 가능 여부 저장 방식, 재고 수량, 품절 처리는 확정하지 않는다.

### 구독

구독은 회원이 특정 SKU를 정해진 수량과 배송 주기로 반복해서 받기 위해 만든 약속이다.

전체 제품 개념에서는 구독이 하나 이상의 상품 또는 SKU를 포함할 가능성을 배제하지 않는다. 첫 번째 수직 MVP에서는 하나의 구독이 하나의 SKU만 대상으로 한다.

복수 SKU 구독은 Post-MVP 범위다.

구독은 구독 회원, 대상 SKU, 선택 수량, 배송 주기, 다음 주문 예정일을 설명해야 한다.

### 배송 주기

배송 주기는 회원이 구독 생성 과정에서 선택하는 반복 간격이며, 다음 주문 예정일을 결정하는 입력이다.

첫 번째 수직 MVP의 배송 주기는 2주, 4주, 8주다.

사용자는 임의의 배송 주기를 입력하지 않는다. 한 가지 배송 주기만 제공하는 방식은 채택하지 않는다.

배송 주기의 코드 표현 방식, enum, 값 객체와 DB 저장 방식은 확정하지 않는다.

### 다음 주문 예정일

다음 주문 예정일은 구독을 바탕으로 다음 정기 주문이 생성될 예정인 날짜를 사용자에게 보여주는 제품 개념이다.

첫 번째 수직 MVP의 다음 주문 예정일은 다음 규칙으로 계산한다.

```text
다음 주문 예정일 = 구독 생성일 + 회원이 선택한 배송 주기
```

제품 규칙은 다음과 같다.

- 사용자는 최초 주문 예정일을 선택하지 않는다.
- 날짜 기준 시간대는 Asia/Seoul이다.
- 날짜 단위로 관리한다.
- 휴일, 주말과 영업일 보정은 하지 않는다.
- 첫 번째 수직 MVP에는 주문 예정일만 포함한다.
- 실제 정기 주문 자동 생성은 범위에서 제외한다.

정확한 코드 계산 책임, 타입, 값 객체와 영속성 방식은 DOMAIN-001에서 구체화한다.

### 주문 예정일과 배송 예정일

두 용어는 다음과 같이 구분한다.

- 주문 예정일: 구독을 바탕으로 다음 정기 주문이 생성될 예정인 날짜
- 배송 예정일: 생성된 주문의 상품이 출고되거나 도착할 것으로 예상되는 날짜

첫 번째 수직 MVP에는 다음 주문 예정일만 포함한다.

실제 주문 생성과 실제 배송 처리가 없으므로 배송 예정일을 화면이나 결과로 제공하지 않는다.

## 도메인 간 제품 관계

- 회원은 상품을 탐색한다.
- 상품은 하나 이상의 상품 옵션 또는 SKU를 제공할 수 있다.
- 회원은 상품의 구독 가능한 SKU를 선택한다.
- 회원은 선택한 SKU 하나, 수량, 배송 주기로 구독을 생성한다.
- 구독은 회원, SKU, 수량, 배송 주기와 다음 주문 예정일을 설명한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.

이 관계는 제품 개념 사이의 관계이며 ERD, 외래 키, JPA 연관관계로 확정하지 않는다.

## 대표 사용자 흐름

```text
회원 로그인
→ 상품 탐색
→ 구독 가능한 SKU 선택
→ 수량과 배송 주기 선택
→ 구독 생성
→ 자신의 구독 확인
→ 다음 주문 예정일 확인
```

## DOMAIN-001에서 구체화할 항목

- 전역 구독 용어와 첫 번째 MVP 단일 SKU 제한의 관계
- 구독 가능 여부의 도메인 표현
- 배송 주기의 코드 표현 방식
- 다음 주문 예정일 계산 책임
- 날짜 값 객체 필요성

- 구독 생성 불변 조건

## 제외 범위

- 일반 구매 주문 생성
- 장바구니
- 여러 상품을 한 번에 주문하는 기능
- 정기 주문 자동 생성
- 실제 결제와 PG 연동
- 결제 실패와 결제 재시도
- 재고 예약과 재고 차감
- 품절 후속 처리
- 실제 배송 처리와 배송 추적
- 배송 예정일 제공
- 구독 수량 변경
- 다음 회차 건너뛰기
- 구독 일시정지, 재개, 해지

- 여러 SKU 묶음 구독
- 운영자 기능과 관리자 화면

## 중단 조건

- 일반 구매, 주문, 결제, 재고, 실제 배송을 반드시 첫 번째 MVP에 포함해야 한다는 승인 문서가 발견되는 경우
- 하나의 구독에 여러 SKU를 포함해야 한다는 승인 결정이 첫 번째 수직 MVP 범위에 대해 발견되는 경우
- 배송 주기 2주, 4주, 8주 또는 다음 주문 예정일 계산 규칙과 충돌하는 승인 결정이 발견되는 경우
- Product Decision이 요구사항 또는 구현 규칙으로 확정되지 않으면 후속 작업을 진행할 수 없는 경우
