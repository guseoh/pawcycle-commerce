# PS-001 PO → BE 인수인계

## 전달 목적

첫 번째 MVP의 제품 도메인 범위와 대표 사용자 흐름을 DOMAIN-001 및 Backend Engineer 역할이 추측 없이 이어받도록 전달한다.

이 문서는 도메인 상세 설계 산출물이 아니다. 엔티티, 테이블, API, 클래스 이름, 상태 전이, 불변식은 이 문서에서 확정하지 않는다.

## 확정된 1차 MVP 범위

회원이 판매 가능한 반려동물 소모품의 상품 옵션 또는 SKU를 선택하고, 수량과 배송 주기를 지정해 정기배송 구독을 생성한 뒤, 자신의 구독과 다음 주문 예정일을 확인한다.

일반 구매는 상위 제품 방향으로 유지하지만 첫 번째 수직 MVP 구현 범위에서는 제외한다.

## 포함 도메인

- 회원
- 상품
- 상품 옵션 또는 SKU
- 구독
- 배송 주기
- 다음 주문 예정일

배송 주기와 다음 주문 예정일은 현재 단계에서 별도 엔티티가 아니라 구독을 설명하는 제품 개념 또는 값이다.

## 도메인별 제품 의미

### 회원

- 첫 번째 MVP의 사용자는 로그인한 회원이다.
- 회원은 상품을 탐색한다.
- 회원은 특정 SKU를 대상으로 구독을 생성한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.
- 비회원 구독과 다른 회원의 구독 조회는 제외한다.

### 상품

상품은 사용자에게 하나의 반려동물 소모품으로 인식되는 공통 상품 정보다. 상품명과 설명 등 공통 정보가 있으며, 하나 이상의 판매 옵션을 가질 수 있다.

### 상품 옵션 또는 SKU

상품 옵션 또는 SKU는 용량, 구성, 맛 등 선택 항목이 확정된 실제 판매 단위다. 첫 번째 MVP에서 회원은 상품 자체가 아니라 구체적인 SKU를 구독 대상으로 선택한다.

### 구독

구독은 회원이 특정 SKU를 정해진 수량과 배송 주기로 반복해서 받기 위해 만든 약속이다. 구독은 구독 회원, 대상 SKU, 선택 수량, 배송 주기, 다음 주문 예정일을 설명해야 한다.

### 배송 주기

배송 주기는 회원이 구독 생성 과정에서 선택하는 반복 간격이며, 다음 주문 예정일을 결정하는 입력이다. 제공할 주기 선택지는 아직 미확정 Product Decision이다.

### 다음 주문 예정일

다음 주문 예정일은 구독에 따라 다음 주문이 생성될 예정인 날짜를 사용자에게 보여주는 제품 개념이다. 정확한 계산식과 책임은 후속 작업에서 결정한다.

## 도메인 간 제품 관계

- 회원은 상품을 탐색한다.
- 상품은 하나 이상의 상품 옵션 또는 SKU를 제공할 수 있다.
- 회원은 상품의 SKU를 선택한다.
- 회원은 선택한 SKU, 수량, 배송 주기로 구독을 생성한다.
- 구독은 회원, SKU, 수량, 배송 주기와 다음 주문 예정일을 설명한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.

이 관계는 제품 개념 사이의 관계이며 ERD, 외래 키, JPA 연관관계로 확정하지 않는다.

## 대표 사용자 흐름

```text
회원
→ 상품 탐색
→ 상품 옵션 또는 SKU 선택
→ 수량과 배송 주기 선택
→ 구독 생성
→ 자신의 구독 확인
→ 다음 주문 예정일 확인
```

## 확정된 Product Decision

| 항목 | 결정 |
| --- | --- |
| 상위 제품 영역 | 반려동물 소모품 이커머스와 정기배송 |
| 1차 MVP 사용자 | 로그인한 회원 |
| 상품 선택 구조 | 상품을 탐색하고 상품 옵션 또는 SKU를 선택 |
| 구독 대상 | 상품 옵션 또는 SKU |
| 구독 단위 | 하나의 구독에 하나의 SKU |
| 구독 입력 | SKU, 수량, 배송 주기 |
| 구독 결과 | 자신의 구독과 다음 주문 예정일 확인 |
| 일반 구매 구현 | 1차 MVP 제외 |
| 주문·결제·재고·배송 구현 | 1차 MVP 제외 |

## 미확정 Product Decision

- 배송 주기를 2주, 4주, 8주로 제한할지
- 첫 번째 MVP에서 한 가지 배송 주기만 제공할지
- 사용자가 최초 주문 예정일을 선택할지
- 구독 생성일을 기준으로 예정일을 자동 계산할지
- 주문 예정일과 배송 예정일을 구분할지
- SKU의 판매 가능 상태를 어떤 제품 용어로 표현할지
- 첫 번째 MVP 상품 범위를 사료 등 특정 카테고리로 제한할지

## 요구사항 정의서 작성 시 결정할 항목

- 상품 목록에서 회원이 확인할 정보
- 상품 상세에서 회원이 확인할 정보
- SKU 선택 조건
- 구독 수량 입력 범위
- 배송 주기 선택지
- 구독 생성 성공 조건
- 구독 생성 실패 조건
- 다음 주문 예정일 표시 규칙
- 자신의 구독 조회 범위
- 비회원 접근 처리
- 다른 회원의 구독 접근 처리

## DOMAIN-001에서 구체화할 항목

- 회원, 상품, SKU와 구독의 책임
- 상품과 SKU의 관계
- 구독 상태가 필요한지
- 구독 생성의 불변 조건
- 판매 불가능한 SKU 처리
- 배송 주기 표현 방식
- 다음 주문 예정일 계산 책임
- 날짜 관련 값 객체가 필요한지

## 제외 범위

- 일반 구매 주문 생성
- 장바구니
- 여러 상품을 한 번에 주문하는 기능
- 정기 주문 자동 생성
- 실제 결제와 PG 연동
- 결제 실패와 결제 재시도
- 재고 예약과 재고 차감
- 품절 후속 처리
- 실제 배송 처리와 배송 추적
- 구독 수량 변경
- 다음 회차 건너뛰기
- 구독 일시정지, 재개, 해지
- 가격 변경 정책
- 구독 할인, 쿠폰, 포인트
- 추천 시스템
- 여러 SKU 묶음 구독
- 운영자 기능과 관리자 화면

## 중단 조건

- 일반 구매, 주문, 결제, 재고, 실제 배송을 반드시 첫 번째 MVP에 포함해야 한다는 승인 문서가 발견되는 경우
- 하나의 구독에 여러 SKU를 포함해야 한다는 승인 결정이 발견되는 경우
- 배송 주기 또는 다음 주문 예정일을 별도 엔티티로 확정해야 한다는 승인 결정이 발견되는 경우
- Product Decision이 요구사항 또는 구현 규칙으로 확정되지 않으면 후속 작업을 진행할 수 없는 경우
