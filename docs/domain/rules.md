# 도메인 규칙(Domain Rules)

이 문서는 승인된 도메인 규칙을 기록한다. 상세 구현, API 계약과 데이터베이스 설계는 해당 작업에서 별도로 확정한다.

## 현재 전역 규칙

- 가격(Pricing), 할인(Discount), 재고(Stock), 구독 가능 여부(Subscription Eligibility), 결제(Payment), 주문 상태(Order State)의 기준은 백엔드다.
- 프론트엔드는 서버가 제공한 결과를 표시하고 사용자 편의 입력 검사를 할 수 있지만 서버 비즈니스 정책을 중복 구현하지 않는다.
- 중복 요청이 중복 주문, 결제, 건너뛰기, 일시정지, 재개, 해지를 만들 수 있는 구독 동작은 멱등성(Idempotency)을 고려해야 한다.
- 상품 정보가 구매 이후 바뀔 수 있으므로 주문 이력에는 스냅숏(Snapshot)이 필요할 수 있다. 정확한 스냅숏 필드는 후속 도메인 설계에서 승인한다.
- 성능 개선은 최적화 전에 측정 근거를 필요로 한다.

| DOMAIN-001 | 구독 수량은 1~10이다. | PS-002 | BE, API, QA | 1 미만 또는 10 초과 수량으로 구독이 생성되지 않음 |

| DOMAIN-001 | 휴일, 주말과 영업일 보정은 적용하지 않는다. | PS-001, PS-002 | BE, API, QA | 보정 없이 배송 주기만 더한 날짜가 산출 |

| DOMAIN-001 | 재고, 품절과 일반 판매 상태는 이번 구독 생성 조건에 연결하지 않는다. | PS-001, PS-002 | BE, API, QA | 구독 생성 조건은 SKU 구독 가능 여부만 사용 |
| DOMAIN-001 | 다음 주문 예정일은 표시용 예정 정보이며 실제 정기 주문 자동 생성은 제외한다. | PS-001, PS-002 | BE, API, QA, UX | 예정일 도래만으로 주문 생성 책임을 만들지 않음 |

## Product Decision이 필요한 정책 영역

- 구독 변경 가능 마감 시점
- 건너뛰기 제한과 동작
- 일시정지와 재개 규칙
- 해지 정책
- 결제 실패 재시도 정책
- 재고 부족 처리
- 가격과 할인 정책
- 환불 또는 보상 정책
- 복수 SKU 구독 정책
- 정기 주문 자동 생성 정책

## 규칙 갱신 방식

새 규칙에는 다음을 포함한다.

- 작업 ID
- 규칙 문장
- 승인 출처
- 영향을 받는 역할
- 테스트 또는 검증 기대 사항
