# 도메인 규칙(Domain Rules)

이 문서는 승인된 도메인 규칙을 기록한다. Phase 0에서는 상세한 커머스(Commerce), 구독(Subscription), 결제(Payment), 재고(Stock), 해지(Cancellation) 정책을 승인하지 않는다.

## 현재 전역 규칙

- 가격(Pricing), 할인(Discount), 재고(Stock), 구독 가능 여부(Subscription Eligibility), 결제(Payment), 주문 상태(Order State)의 기준은 백엔드다.
- 프론트엔드는 서버가 제공한 결과를 표시하고 사용자 편의 입력 검사를 할 수 있지만 서버 비즈니스 정책을 중복 구현하지 않는다.
- 중복 요청이 중복 주문, 결제, 건너뛰기, 일시정지, 재개, 해지를 만들 수 있는 구독 동작은 멱등성(Idempotency)을 고려해야 한다.
- 상품 정보가 구매 이후 바뀔 수 있으므로 주문 이력에는 스냅숏(Snapshot)이 필요할 수 있다. 정확한 스냅숏 필드는 도메인 설계에서 승인한다.
- 성능 개선은 최적화 전에 측정 근거를 필요로 한다.

## Product Decision이 필요한 정책 영역

- 지원할 배송 주기
- 다음 배송일 계산 규칙
- 구독 변경 가능 마감 시점
- 건너뛰기 제한과 동작
- 일시정지와 재개 규칙
- 해지 정책
- 결제 실패 재시도 정책
- 재고 부족 처리
- 가격과 할인 정책
- 환불 또는 보상 정책

## 규칙 갱신 방식

새 규칙에는 다음을 포함한다.

- 작업 ID
- 규칙 문장
- 승인 출처
- 영향을 받는 역할
- 테스트 또는 검증 기대 사항
