# MVP 범위

## 문서 상태

- 작업 ID: `PS-001`
- 역할: Product Planner
- 상태: PS-001 기준 제품 결정안
- 범위 제한: 제품 정책과 MVP 경계를 정의한다. 도메인 모델, 상태 전이, 테이블, API, 화면 설계는 정의하지 않는다.

## MVP 목표

PawCycle Commerce MVP의 목표는 반려견/반려묘 보호자가 소모품을 단건으로 구매하거나 정기 구독으로 반복 구매할 수 있는 최소 제품 경험을 제공하는 것이다. MVP는 구독 커머스의 핵심 사용자 행동과 예외 상황을 검증하는 데 집중한다.

## 포함 범위

| 영역 | MVP 포함 | PS-001 기준 결정 |
| --- | --- | --- |
| 대상 반려동물 | 반려견, 반려묘 | 다른 동물군은 제외한다. |
| 상품군 | 사료, 간식, 고양이 모래, 배변 패드, 탈취제, 샴푸, 기타 위생 소모품 | 대표 SKU 중심으로 시작한다. |
| 판매 단위 | SKU | 사용자는 크기, 맛, 포장 단위 등으로 구분되는 SKU를 선택해 구매한다. |
| 단건 구매 | 포함 | 사용자는 카트에 여러 SKU를 담아 단건 주문할 수 있다. |
| 정기 구독 | 포함 | 하나의 구독은 하나의 SKU를 대상으로 한다. 사용자는 여러 구독을 만들 수 있다. |
| 배송 주기 | 2주, 4주, 8주 | 자유 주기 입력은 제외한다. |
| 첫 배송일 | 자유 선택 제외 | 첫 회차는 구독 생성 시점의 가장 빠른 처리 기준으로 잡고, 이후 회차부터 선택한 주기를 따른다. |
| 구독 수량 변경 | 포함 | 다음 회차 생성 마감 전까지 미래 회차에 적용한다. |
| 다음 회차 건너뛰기 | 포함 | 한 번의 예정 회차를 건너뛰고 이후 주기를 유지한다. |
| 일시정지와 재개 | 포함 | 일시정지 중에는 새 회차가 진행되지 않으며, 재개 후 다음 가능한 회차부터 다시 진행한다. |
| 구독 해지 | 포함 | 해지한 구독은 재개할 수 없고, 필요하면 새 구독을 만든다. |
| 변경 마감 시점 | 포함 | 다음 회차 생성 예정일 전날 23:59 KST까지 변경할 수 있다. |
| 가격 정책 | 포함 | 정기 주문 가격은 회차 생성 시점의 현재 가격을 기준으로 하며 주문에 기록된 가격은 유지한다. |
| 가격 변경 고지 | 포함 | MVP에서는 서비스 화면에서 다음 회차 전 현재 가격을 확인하게 한다. 이메일, 문자, 푸시 고지는 제외한다. |
| 품절 처리 | 포함 | 품절이면 해당 회차 진행을 보류하고 사용자에게 화면에서 알린다. 대체 상품, 부분 배송, 자동 보상은 제외한다. |
| 판매 중단 처리 | 포함 | 판매 중단 SKU는 신규 구매와 신규 구독을 막고, 기존 이력이나 기존 구독 확인에는 노출한다. |
| 결제 | 모의 결제 | 성공과 실패를 확인할 수 있는 수준으로 제공한다. 실제 PG 연동은 제외한다. |
| 결제 실패 | 포함 | 자동 재시도는 MVP에서 하지 않는다. 사용자는 실패한 회차를 확인하고 수동으로 다시 시도한다. |
| 결제수단 변경 | 제외 | 실제 카드 변경, 카드 보관, PG 토큰 관리는 MVP 이후로 넘긴다. |
| 주문 취소 | 부분 포함 | 결제 완료 후 배송 준비 전까지 사용자가 취소할 수 있다. 실제 환불, 교환, 반품은 제외한다. |
| 배송 정책 | 단일 정책 | 단건 주문과 정기 주문에 같은 기본 배송 정책을 적용한다. 무료배송 조건, 지역별 추가요금, 배송 추적은 제외한다. |
| 운영자 기능 | 최소 포함 | 상품 판매 가능 여부, 재고, 주문, 구독 현황을 확인하고 조정하는 최소 운영 기능을 포함한다. |

## 제외 범위

다음 항목은 MVP 범위에서 제외한다.

- 실제 PG 연동, 카드 저장, 결제수단 변경
- 쿠폰, 포인트, 적립금, 정기 구독 할인
- 리뷰, 평점, Q&A, 커뮤니티
- 반려동물 프로필 기반 추천
- 첫 배송일 자유 선택, 사용자가 직접 입력하는 배송 주기
- 하나의 구독에 여러 SKU를 묶는 기능
- 자동 결제 재시도 정책
- 이메일, 문자, 푸시, 카카오 알림
- 배송 추적, 택배사 연동, 지역별 배송 제한
- 실제 환불, 교환, 반품
- 다중 판매자, 입점사 정산, 권한별 관리자
- 고급 검색, 개인화 추천, 번들 상품

| 구독 해지 후 재개 | 해지한 구독은 재개하지 않는다. | 없음 |

| 품절 | 해당 회차 진행을 보류하고 화면에서 사용자에게 알린다. | 운영자 조치 흐름은 이후 작업에서 구체화한다. |
| 결제 실패 | 자동 재시도 없이 사용자가 수동 재시도한다. 구독 자체는 유지한다. | 실제 결제수단 변경은 MVP 이후 결정한다. |
| 주문 취소 | 배송 준비 전까지만 사용자 취소를 허용한다. | 배송 준비 판단은 DOMAIN-001에서 다룬다. |
| 배송비 | 단일 기본 배송 정책을 사용한다. | 금액과 무료배송 여부는 MVP 설정값으로 별도 확정한다. |
| 관리자 UI | 최소 운영 기능은 필요하지만 별도 고급 관리자 제품은 제외한다. | 화면 형태는 FE/UX 작업에서 결정한다. |

## MVP 완료 기준

MVP는 다음 조건을 만족해야 한다.

1. 사용자가 반려견/반려묘 소모품을 탐색하고 단건 주문을 완료할 수 있다.
2. 사용자가 단일 SKU를 대상으로 2주, 4주, 8주 중 하나의 구독을 만들 수 있다.
3. 사용자가 다음 회차 전에 수량 변경, 건너뛰기, 일시정지, 재개, 해지를 수행할 수 있다.
4. 사용자가 품절과 결제 실패의 이유를 확인하고 가능한 다음 행동을 이해할 수 있다.
5. 운영자가 MVP 운영에 필요한 상품, 재고, 주문, 구독 현황을 확인할 수 있다.
