# 사용자 여정

## 문서 상태

- 작업 ID: `PS-001`
- 역할: Product Planner
- 상태: PS-001 기준 제품 결정안
- 범위 제한: 사용자 행동과 기대 결과만 정의한다. 화면 설계, API, 데이터 구조, 상태 전이는 정의하지 않는다.

## 여정 1. 단건 구매

### 사용자

반려견/반려묘 소모품을 지금 구매하려는 보호자

### 목표

필요한 SKU를 찾아 카트에 담고 단건 주문을 완료한다.

### 주요 흐름

1. 사용자는 상품 목록에서 반려견/반려묘 소모품을 탐색한다.
2. 사용자는 상품 상세에서 SKU, 가격, 판매 가능 여부를 확인한다.
3. 사용자는 하나 이상의 SKU를 카트에 담는다.
4. 사용자는 배송지와 모의 결제 결과를 확인하고 주문을 완료한다.
5. 사용자는 주문 내역에서 주문 결과를 확인한다.

### 기대 결과

사용자는 정기 구독을 만들지 않고도 필요한 상품을 바로 구매할 수 있다. 품절 또는 판매 중단 SKU는 구매 가능한 상품처럼 보이지 않아야 한다.

## 여정 2. 정기 구독 생성

### 사용자

같은 소모품을 반복 구매하는 보호자

### 목표

반복 구매할 SKU를 선택해 정기 구독을 만든다.

### 주요 흐름

1. 사용자는 상품 상세에서 정기 구독 가능 여부를 확인한다.
2. 사용자는 SKU, 수량, 배송 주기 2주, 4주, 8주 중 하나를 선택한다.
3. 사용자는 첫 회차가 가장 빠른 처리 기준으로 잡히고 이후 회차가 선택 주기를 따른다는 점을 확인한다.
4. 사용자는 모의 결제 정보를 확인하고 구독을 만든다.
5. 사용자는 구독 목록에서 다음 예정 회차와 현재 가격을 확인한다.

### 기대 결과

사용자는 반복 구매를 자동화할 수 있다. 하나의 구독에는 하나의 SKU만 포함되며, 여러 상품을 반복 구매하려면 여러 구독을 만든다.

## 여정 3. 다음 회차 수량 변경

### 사용자

이번 달 사용량이 달라져 다음 회차 수량을 조정하려는 보호자

### 목표

다음 회차 생성 마감 전까지 수량을 변경한다.

### 주요 흐름

1. 사용자는 구독 상세에서 다음 예정 회차를 확인한다.
2. 사용자는 수량 변경이 다음 회차부터 적용된다는 점을 확인한다.
3. 사용자는 마감 시점 전까지 수량을 변경한다.
4. 사용자는 변경된 수량과 예상 가격을 확인한다.

### 기대 결과

사용자는 이미 처리 중인 회차를 혼동하지 않고, 앞으로 생성될 회차에 적용될 수량을 이해한다.

## 여정 4. 건너뛰기, 일시정지, 재개, 해지

### 사용자

재고가 남았거나 당분간 필요하지 않은 보호자

### 목표

구독을 유지하면서 다음 회차를 건너뛰거나, 구독을 잠시 멈췄다가 다시 시작하거나, 필요 없어진 구독을 해지한다.

### 주요 흐름

1. 사용자는 구독 상세에서 다음 예정 회차를 확인한다.
2. 사용자는 다음 회차만 건너뛰거나 구독을 일시정지한다.
3. 일시정지 중인 사용자는 필요할 때 구독을 재개한다.
4. 더 이상 필요하지 않으면 구독을 해지한다.
5. 사용자는 해지한 구독은 재개할 수 없고 새 구독을 만들어야 한다는 점을 확인한다.

### 기대 결과

사용자는 구독을 불필요하게 취소하지 않고 상황에 맞게 조정할 수 있다. 해지와 일시정지는 사용자에게 명확히 구분되어야 한다.

## 여정 5. 결제 실패 확인과 수동 재시도

### 사용자

정기 회차 결제가 실패한 보호자

### 목표

결제 실패 이유를 확인하고 수동으로 다시 시도한다.

### 주요 흐름

1. 사용자는 주문 또는 구독 화면에서 결제 실패를 확인한다.
2. 사용자는 해당 회차가 배송으로 진행되지 않는다는 점을 확인한다.
3. 사용자는 모의 결제를 수동으로 다시 시도한다.
4. 성공하면 해당 회차가 다시 진행 가능한 상태임을 확인한다.

### 기대 결과

구독 자체는 유지되지만 결제에 실패한 회차는 배송되지 않는다. MVP에서는 자동 재시도와 실제 카드 변경을 제공하지 않는다.

## 여정 6. 품절 또는 판매 중단 확인

### 사용자

구독 중인 SKU가 품절되었거나 판매 중단된 보호자

### 목표

이번 회차가 왜 진행되지 않는지 이해하고 다음 행동을 결정한다.

### 주요 흐름

1. 사용자는 구독 또는 주문 화면에서 품절 또는 판매 중단 안내를 확인한다.
2. 품절인 경우 사용자는 해당 회차가 보류되었음을 확인한다.
3. 판매 중단인 경우 사용자는 신규 구매 또는 신규 구독이 불가능하다는 점을 확인한다.
4. 사용자는 구독 해지 또는 다른 SKU 구독 생성을 선택할 수 있다.

### 기대 결과

사용자는 실패 원인을 추측하지 않아도 된다. MVP에서는 대체 상품 자동 추천, 부분 배송, 보상 처리를 제공하지 않는다.

## 여정 7. 최소 운영 확인

### 사용자

단일 판매자 또는 운영자

### 목표

MVP 운영에 필요한 상품, 재고, 주문, 구독 현황을 확인하고 기본 조치를 수행한다.

### 주요 흐름

1. 운영자는 상품의 판매 가능 여부와 재고를 확인한다.
2. 운영자는 품절 또는 판매 중단이 고객 구매와 구독에 어떻게 보이는지 확인한다.
3. 운영자는 주문과 구독 현황에서 결제 실패, 품절, 해지 요청을 확인한다.
4. 운영자는 MVP 운영에 필요한 최소 조치를 수행한다.

### 기대 결과

운영자는 복잡한 관리자 제품 없이도 MVP 검증에 필요한 기본 운영 상태를 확인할 수 있다.
