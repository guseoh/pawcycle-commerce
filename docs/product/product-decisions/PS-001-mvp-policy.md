# PS-001 제품 결정 기록

## 문서 상태

- 작업 ID: `PS-001`
- 역할: Product Planner
- 상태: PS-001 기준 제품 결정안
- 목적: DOMAIN-001과 후속 역할이 제품 범위와 남은 결정을 구분할 수 있도록 기록한다.
- 범위 제한: 이 문서는 제품 정책 기록이며, 도메인 모델, 상태 전이, API, 데이터 구조를 정의하지 않는다.

## 결정 기준

PS-001에서는 추천안을 승인된 외부 계약처럼 표현하지 않는다. 아래 항목은 이번 작업에서 후속 설계 입력으로 사용하는 제품 기준이며, PR 리뷰에서 조정될 수 있다.

## 명시적 Post-MVP 후보

- 실제 PG 연동과 결제수단 변경
- 자동 결제 재시도
- 가격 변경 이메일, 문자, 푸시, 카카오 알림
- 첫 배송일 자유 선택
- 하나의 구독에 여러 SKU를 묶는 기능
- 구독 할인, 쿠폰, 포인트
- 대체 상품 추천, 부분 배송, 보상 정책
- 배송 추적과 택배사 연동
- 다중 판매자와 정산

## 이전 PS-001 브랜치 참고 결과

기존 `spec/PS-001-service-vision-mvp` 브랜치는 제품 문제와 MVP 후보를 참고했다. 다만 해당 브랜치는 저장소 부트스트랩 전 상태를 다수 포함해 현재 `main`의 문서, 스크립트, 자동화 파일을 되돌리는 변경이 섞여 있으므로 병합하거나 체리픽하지 않았다.

재사용한 내용은 다음과 같다.

- 반려견/반려묘 소모품 반복 구매 문제 정의
- 단건 구매와 정기 구독을 함께 제공하는 방향
- SKU 기반 구독, 2주/4주/8주 주기, 단일 판매자 운영 전제
- 실제 PG, 다중 판매자, 리뷰, 쿠폰, 추천, 배송 추적 제외 방향

조정한 내용은 다음과 같다.

- 구독 관리 범위를 수량 변경, 건너뛰기, 일시정지, 재개, 해지로 명확히 확장했다.
- 결제 실패는 자동 재시도 없이 수동 재시도로 정리했다.
- 가격 변경, 품절, 판매 중단, 주문 취소, 배송 정책을 제품 정책으로 별도 기록했다.
