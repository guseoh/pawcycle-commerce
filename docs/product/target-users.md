# 대상 사용자

## 문서 상태

- 작업 ID: `PS-001`
- 역할: Product Planner
- 상태: PS-001 기준 제품 결정안
- 범위 제한: 사용자 유형, 문제, 기대 행동만 정의한다. 권한 모델, 데이터 구조, 인증 방식은 정의하지 않는다.

## 1차 사용자

### 반려견/반려묘 보호자

반려견 또는 반려묘를 키우며 사료, 간식, 배변용품, 위생용품을 반복 구매하는 고객이다. MVP의 핵심 사용자는 같은 상품을 일정 주기로 다시 구매하거나, 특정 소모품이 떨어지기 전에 미리 받고 싶은 사람이다.

주요 요구는 다음과 같다.

- 단건 구매로 필요한 상품을 바로 주문한다.
- 자주 쓰는 SKU를 정기 구독으로 등록한다.
- 다음 회차 전에 수량을 조정하거나 한 번 건너뛴다.
- 당분간 필요 없을 때 구독을 일시정지하고 필요할 때 재개한다.
- 더 이상 필요 없는 구독을 해지한다.
- 품절, 판매 중단, 결제 실패로 진행되지 않는 이유를 이해한다.

## 2차 사용자

### 단일 판매자 또는 운영자

MVP는 단일 판매자 운영을 전제로 한다. 운영자는 상품 판매 가능 여부와 재고를 관리하고, 고객 주문 및 구독 현황을 확인하는 최소 역할을 가진다.

주요 요구는 다음과 같다.

- MVP 카탈로그에 노출할 반려견/반려묘 소모품을 관리한다.
- 판매 가능, 품절, 판매 중단 상태를 고객 경험에 맞게 반영한다.
- 주문 및 구독에서 발생한 품절, 결제 실패, 해지 요청을 확인한다.
- 복잡한 정산, 다중 판매자 운영, 고객센터 워크플로는 MVP 이후로 넘긴다.

## MVP 대상 상품군

MVP는 반려견/반려묘 소모품에 한정한다.

포함 상품군은 다음과 같다.

- 사료
- 간식
- 고양이 모래
- 배변 패드
- 탈취제, 샴푸, 기타 위생 소모품

MVP 카탈로그는 모든 상품군을 넓게 확보하기보다 대표 SKU로 반복 구매 흐름을 검증하는 데 초점을 둔다.

## MVP 비대상 사용자

다음 사용자는 MVP의 직접 대상이 아니다.

- 다중 판매자, 입점 브랜드, 정산 담당자
- 병원, 미용실, 훈련소 등 오프라인 파트너
- 반려동물 건강 기록이나 상담 서비스를 찾는 사용자
- 쿠폰, 포인트, 리뷰, 커뮤니티 중심 쇼핑 경험을 기대하는 사용자
- 해외 배송 또는 복잡한 지역별 배송 정책이 필요한 사용자

## 사용자 경험 원칙

1. 반복 구매는 사용자가 제어할 수 있어야 한다.
2. 정기 구독 변경은 앞으로 생성될 회차에만 적용된다는 점을 명확히 보여준다.
3. 품절, 결제 실패, 판매 중단은 숨기지 않고 사용자에게 이유와 가능한 다음 행동을 보여준다.
4. 단건 구매와 정기 구독은 서로 대체재가 아니라 같은 카탈로그에서 선택 가능한 구매 방식으로 제공한다.
