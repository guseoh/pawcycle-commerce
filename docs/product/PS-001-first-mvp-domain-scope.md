# PS-001 1차 MVP 도메인 범위

## 작업 목적

PS-001은 PawCycle Commerce의 첫 번째 수직 MVP에서 실제로 다룰 제품 도메인과 대표 사용자 흐름을 확정한다.

이 문서는 후속 요구사항 정의와 DOMAIN-001 도메인 상세 설계의 입력이다. 상세 기능 요구사항, 인수 조건, ERD, API, DB 구조, 애플리케이션 구현 방식은 이 문서에서 작성하거나 확정하지 않는다.

## 상위 제품 방향

PawCycle Commerce의 상위 제품 방향은 다음으로 유지한다.

- 반려동물 소모품
- 일반 구매 이커머스
- 반복 구매
- 정기배송 구독

이번 작업은 반려동물 소모품이라는 산업 영역이나 전체 서비스 도메인을 다시 선택하지 않는다. 다만 일반 구매는 상위 제품 방향으로만 유지하고, 첫 번째 수직 MVP의 구현 범위에서는 제외한다.

## 1차 MVP 목표

회원이 판매 가능한 반려동물 소모품의 상품 옵션 또는 SKU를 선택하고, 수량과 배송 주기를 지정해 정기배송 구독을 생성한 뒤, 자신의 구독과 다음 주문 예정일을 확인한다.

수량은 별도 도메인으로 분리하지 않고 구독 생성 시 선택하는 정보로 다룬다.

## 주요 사용자

첫 번째 MVP의 사용자는 로그인한 회원이다.

비회원 구독, 운영자 기능, 관리자 화면, 다른 회원의 구독 조회는 첫 번째 MVP 범위에 포함하지 않는다.

## 포함 도메인

첫 번째 MVP는 다음 도메인과 제품 개념으로 제한한다.

- 회원
- 상품
- 상품 옵션 또는 SKU
- 구독
- 배송 주기
- 다음 주문 예정일

위 항목을 모두 독립적인 엔티티로 확정하지 않는다. 특히 배송 주기와 다음 주문 예정일은 현재 단계에서 구독을 설명하는 제품 개념 또는 값으로만 다룬다.

## 도메인별 제품 의미

### 회원

- 첫 번째 MVP의 사용자는 로그인한 회원이다.
- 회원은 상품을 탐색한다.
- 회원은 특정 SKU를 대상으로 구독을 생성한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.
- 비회원 구독은 첫 번째 MVP에서 제외한다.
- 회원은 다른 회원의 구독을 볼 수 없다.

로그인 구현 방식, 회원 테이블, 인증 토큰, 세션 구조, Spring Security 설정, 권한 클래스는 설계하지 않는다.

### 상품

상품은 사용자에게 하나의 반려동물 소모품으로 인식되는 공통 상품 정보다.

- 상품명과 설명 등 공통 정보가 있다.
- 상품 하나는 하나 이상의 판매 옵션을 가질 수 있다.
- 회원은 상품을 탐색한 뒤 구체적인 옵션을 선택한다.

상품 테이블, 상품 컬럼, 상품 Entity, 상품 API는 설계하지 않는다.

### 상품 옵션 또는 SKU

상품은 사용자에게 보여주는 공통 상품 정보이고, 상품 옵션 또는 SKU는 용량, 구성, 맛 등 선택 항목이 확정된 실제 판매 단위다.

예시는 다음과 같다.

- 상품: 연어 사료
- 상품 옵션 또는 SKU: 2kg, 5kg, 10kg

첫 번째 MVP에서 회원은 상품 자체가 아니라 구체적인 SKU를 구독 대상으로 선택한다.

SKU 코드 생성 규칙, 가격 컬럼, 재고 컬럼, 옵션 조합 테이블, Product와 SKU의 JPA 연관관계는 설계하지 않는다.

### 구독

구독은 회원이 특정 SKU를 정해진 수량과 배송 주기로 반복해서 받기 위해 만든 약속이다.

이번 단계에서는 구독에 다음 개념이 필요하다는 수준만 정리한다.

- 구독 회원
- 구독 대상 SKU
- 선택 수량
- 배송 주기
- 다음 주문 예정일

구독 Entity, 구독 테이블, 구독 상태 enum, 상태 전이, 불변 조건의 구현 방식, 외래 키는 설계하지 않는다.

### 배송 주기

배송 주기는 회원이 상품을 반복해서 받을 간격이다.

- 회원이 구독 생성 과정에서 선택한다.
- 구독의 반복 간격을 표현한다.
- 다음 주문 예정일을 결정하는 입력이 된다.

배송 주기를 2주, 4주, 8주처럼 고정 선택지로 제공할지, 첫 번째 MVP에서 한 가지 주기만 제공할지, 사용자가 임의의 일수를 입력할 수 있는지는 아직 확정하지 않는다.

### 다음 주문 예정일

다음 주문 예정일은 구독에 따라 다음 주문이 생성될 예정인 날짜를 사용자에게 보여주는 제품 개념이다.

- 구독 생성 결과에서 확인할 수 있다.
- 자신의 구독 조회에서 확인할 수 있다.
- 회원이 다음 반복 구매 시점을 이해하는 데 사용한다.

정확한 날짜 계산식, 구독 생성일과의 관계, 최초 배송일 선택 여부, 주문 예정일과 배송 예정일의 구분, 시간대, 휴일 처리, 날짜 변경 정책은 후속 요구사항 정의 또는 도메인 설계 작업으로 넘긴다.

## 도메인 간 제품 관계

- 회원은 상품을 탐색한다.
- 상품은 하나 이상의 상품 옵션 또는 SKU를 제공할 수 있다.
- 회원은 상품의 SKU를 선택한다.
- 회원은 선택한 SKU, 수량, 배송 주기로 구독을 생성한다.
- 구독은 회원, SKU, 수량, 배송 주기와 다음 주문 예정일을 설명한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.

이 관계는 제품 개념 사이의 관계다. ERD 카디널리티, 데이터베이스 외래 키, 테이블 관계, JPA 연관관계, 객체 참조 방향으로 확정하지 않는다.

## 대표 사용자 흐름

```text
회원
→ 상품 탐색
→ 상품 옵션 또는 SKU 선택
→ 수량과 배송 주기 선택
→ 구독 생성
→ 자신의 구독 확인
→ 다음 주문 예정일 확인
```

## MVP 포함 범위

- 로그인한 회원
- 반려동물 소모품 상품 탐색
- 상품별 옵션 또는 SKU 확인
- 하나의 SKU를 대상으로 한 구독
- 구독 수량 선택
- 배송 주기 선택
- 구독 생성
- 자신의 구독 조회
- 다음 주문 예정일 확인

## MVP 제외 범위

- 일반 구매 주문 생성
- 장바구니
- 여러 상품을 한 번에 주문하는 기능
- 정기 주문 자동 생성
- 실제 결제
- PG 연동
- 결제 실패
- 결제 재시도
- 재고 예약
- 재고 차감
- 품절 후속 처리
- 실제 배송 처리
- 배송 추적
- 구독 수량 변경
- 다음 회차 건너뛰기
- 구독 일시정지
- 구독 재개
- 구독 해지
- 가격 변경 정책
- 구독 할인
- 쿠폰
- 포인트
- 추천 시스템
- 여러 SKU 묶음 구독
- 운영자 기능
- 관리자 화면

제외된 기능은 전체 제품 방향에서 삭제하지 않는다. 첫 번째 MVP 이후의 후속 기능 후보로 남긴다.

## 미확정 Product Decision

- 배송 주기를 2주, 4주, 8주로 제한할지
- 첫 번째 MVP에서 한 가지 배송 주기만 제공할지
- 사용자가 최초 주문 예정일을 선택할지
- 구독 생성일을 기준으로 예정일을 자동 계산할지
- 주문 예정일과 배송 예정일을 구분할지
- SKU의 판매 가능 상태를 어떤 제품 용어로 표현할지
- 첫 번째 MVP 상품 범위를 사료 등 특정 카테고리로 제한할지

미확정 내용은 요구사항이나 구현 규칙으로 확정하지 않는다.

## 다음 요구사항 정의 작업의 입력

- 상품 목록에서 회원이 확인할 정보
- 상품 상세에서 회원이 확인할 정보
- SKU 선택 조건
- 구독 수량 입력 범위
- 배송 주기 선택지
- 구독 생성 성공 조건
- 구독 생성 실패 조건
- 다음 주문 예정일 표시 규칙
- 자신의 구독 조회 범위
- 비회원 접근 처리
- 다른 회원의 구독 접근 처리

## DOMAIN-001로 넘길 항목

- 회원, 상품, SKU와 구독의 책임
- 상품과 SKU의 관계
- 구독 상태가 필요한지
- 구독 생성의 불변 조건
- 판매 불가능한 SKU 처리
- 배송 주기 표현 방식
- 다음 주문 예정일 계산 책임
- 날짜 관련 값 객체가 필요한지

엔티티, 테이블, API와 클래스 이름은 확정하지 않는다.
