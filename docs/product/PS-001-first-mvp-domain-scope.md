# PS-001 1차 MVP 도메인 범위

## 작업 목적

PS-001은 PawCycle Commerce의 첫 번째 수직 MVP에서 실제로 다룰 제품 도메인과 대표 사용자 흐름을 확정한다.

이 MVP는 상용 서비스 전체 흐름을 완성하는 단계가 아니라, 구독 생성과 조회에 필요한 최소 제품 흐름과 핵심 도메인 규칙을 검증하는 첫 번째 수직 MVP다.

이 문서는 후속 요구사항 정의와 DOMAIN-001 도메인 상세 설계의 입력이다. 상세 기능 요구사항, 인수 조건, ERD, API, DB 구조, 애플리케이션 구현 방식은 이 문서에서 작성하거나 확정하지 않는다.

## 상위 제품 방향

PawCycle Commerce의 상위 제품 방향은 다음으로 유지한다.

- 반려동물 소모품
- 일반 구매 이커머스
- 반복 구매
- 정기배송 구독

이번 작업은 반려동물 소모품이라는 산업 영역이나 전체 서비스 도메인을 다시 선택하지 않는다. 일반 구매는 상위 제품 방향으로 유지하지만, 첫 번째 수직 MVP의 구현 범위에서는 제외한다.

## 1차 MVP 목표

로그인한 회원이 개와 고양이용 사료 상품을 탐색하고, 구독 가능한 SKU 하나를 선택한 뒤, 수량과 배송 주기를 지정해 정기배송 구독을 생성하고 자신의 구독과 다음 주문 예정일을 확인한다.

수량은 별도 도메인으로 분리하지 않고 구독 생성 시 선택하는 정보로 다룬다.

## 주요 사용자

첫 번째 MVP의 사용자는 로그인한 회원이다.

비회원 구독, 운영자 기능, 관리자 화면, 다른 회원의 구독 조회는 첫 번째 MVP 범위에 포함하지 않는다.

## 상품 예시와 검증 범위

첫 번째 수직 MVP의 상품 예시와 검증 데이터는 개와 고양이용 사료를 중심으로 한다.

예시는 다음과 같다.

- 상품: 연어 사료
- 상품 옵션 또는 SKU: 2kg, 5kg, 10kg

상품과 SKU라는 도메인 개념은 사료에 종속시키지 않는다. 간식, 고양이 모래, 배변 패드, 위생용품은 후속 상품 범위다.

사료 전용 Entity, 클래스, 테이블은 확정하지 않는다.

## 포함 도메인

첫 번째 MVP는 다음 도메인과 제품 개념으로 제한한다.

- 회원
- 상품
- 상품 옵션 또는 SKU
- 구독
- 배송 주기
- 다음 주문 예정일

위 항목을 모두 독립적인 엔티티로 확정하지 않는다. 특히 배송 주기와 다음 주문 예정일은 현재 단계에서 구독을 설명하는 제품 개념 또는 값으로만 다룬다.

## 도메인별 제품 의미

### 회원

- 첫 번째 MVP의 사용자는 로그인한 회원이다.
- 회원은 상품을 탐색한다.
- 회원은 특정 SKU를 대상으로 구독을 생성한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.
- 비회원 구독은 첫 번째 MVP에서 제외한다.
- 회원은 다른 회원의 구독을 볼 수 없다.

로그인 구현 방식, 회원 테이블, 인증 토큰, 세션 구조, Spring Security 설정, 권한 클래스는 설계하지 않는다.

### 상품

상품은 사용자에게 하나의 반려동물 소모품으로 인식되는 공통 상품 정보다.

- 상품명과 설명 등 공통 정보가 있다.
- 상품 하나는 하나 이상의 상품 옵션 또는 SKU를 가질 수 있다.
- 첫 번째 수직 MVP의 예시와 검증 데이터는 개와 고양이용 사료를 중심으로 한다.
- 회원은 상품을 탐색한 뒤 구체적인 SKU를 선택한다.

상품 테이블, 상품 컬럼, 상품 Entity, 상품 API는 설계하지 않는다.

### 상품 옵션 또는 SKU

상품은 사용자에게 보여주는 공통 상품 정보이고, 상품 옵션 또는 SKU는 용량, 구성, 맛 등 선택 항목이 확정된 단위다.

첫 번째 MVP에서 회원은 상품 자체가 아니라 구독 가능한 SKU를 구독 대상으로 선택한다.

판매 상태와 구독 가능 여부는 다른 제품 개념이다.

- 판매 상태: 일반적인 상품 판매 가능 여부
- 구독 가능 여부: 해당 SKU를 정기배송 구독 대상으로 선택할 수 있는지

회원은 구독 가능한 SKU만 구독할 수 있다.

상태 enum 이름, 상태 코드, DB 컬럼, 판매 상태 전이, 구독 가능 여부 저장 방식, 재고 수량, 품절 처리는 확정하지 않는다.

SKU 코드 생성 규칙, 가격 컬럼, 재고 컬럼, 옵션 조합 테이블, Product와 SKU의 JPA 연관관계는 설계하지 않는다.

### 구독

구독은 회원이 특정 SKU를 정해진 수량과 배송 주기로 반복해서 받기 위해 만든 약속이다.

전체 제품 개념에서는 구독이 하나 이상의 상품 또는 SKU를 포함할 가능성을 배제하지 않는다. 다만 첫 번째 수직 MVP에서는 하나의 구독이 하나의 SKU만 대상으로 한다.

복수 SKU 구독은 Post-MVP 범위다.

이번 단계에서는 구독에 다음 개념이 필요하다는 수준만 정리한다.

- 구독 회원
- 구독 대상 SKU
- 선택 수량
- 배송 주기
- 다음 주문 예정일

구독 Entity, 구독 테이블, 구독 상태 enum, 상태 전이, 불변 조건의 구현 방식, 외래 키는 설계하지 않는다.

전역 구독 용어와 첫 번째 MVP의 단일 SKU 제한 사이의 최종 정합성은 DOMAIN-001에서 정리한다.

### 배송 주기

배송 주기는 회원이 상품을 반복해서 받을 간격이다.

- 회원은 구독 생성 시 2주, 4주, 8주 중 하나를 선택한다.
- 첫 번째 수직 MVP에서는 승인된 고정 선택지 외의 배송 주기를 입력할 수 없다.
- SKU별로 서로 다른 배송 주기 선택지를 제한하는 정책은 첫 번째 수직 MVP에서 적용하지 않는다.

사용자가 임의의 일수를 입력하는 방식과 한 가지 배송 주기만 제공하는 방식은 첫 번째 수직 MVP에서 채택하지 않는다.

배송 주기의 코드 표현 방식, enum, 값 객체와 DB 저장 방식은 확정하지 않는다.

### 다음 주문 예정일

다음 주문 예정일은 구독을 바탕으로 다음 정기 주문이 생성될 예정인 날짜를 사용자에게 보여주는 제품 개념이다.

첫 번째 수직 MVP의 다음 주문 예정일은 다음 규칙으로 계산한다.

```text
다음 주문 예정일 = 구독 생성일 + 회원이 선택한 배송 주기
```

예시는 다음과 같다.

```text
구독 생성일: 2026-07-03
선택한 배송 주기: 4주
다음 주문 예정일: 2026-07-31
```

제품 규칙은 다음과 같다.

- 사용자는 최초 주문 예정일을 직접 선택하지 않는다.
- 날짜 단위로 관리한다.
- 제품 기준 시간대는 Asia/Seoul이다.
- 휴일, 주말과 영업일 보정은 하지 않는다.
- 첫 번째 수직 MVP에서는 다음 주문 예정일만 표시한다.
- 다음 배송 예정일은 제공하지 않는다.

다음 주문 예정일은 이번 MVP에서 사용자에게 보여주는 예정 정보다. 해당 날짜에 실제 주문을 자동 생성하는 기능은 이번 범위에 포함하지 않는다.

정확한 코드 계산 책임, 타입, 값 객체와 영속성 방식은 DOMAIN-001에서 구체화한다.

### 주문 예정일과 배송 예정일

두 용어는 다음과 같이 구분한다.

- 주문 예정일: 구독을 바탕으로 다음 정기 주문이 생성될 예정인 날짜
- 배송 예정일: 생성된 주문의 상품이 출고되거나 도착할 것으로 예상되는 날짜

첫 번째 수직 MVP에는 다음 주문 예정일만 포함한다.

실제 주문 생성과 실제 배송 처리가 없으므로 배송 예정일을 화면이나 결과로 제공하지 않는다.

## 도메인 간 제품 관계

- 회원은 상품을 탐색한다.
- 상품은 하나 이상의 상품 옵션 또는 SKU를 제공할 수 있다.
- 회원은 상품의 구독 가능한 SKU를 선택한다.
- 회원은 선택한 SKU 하나, 수량, 배송 주기로 구독을 생성한다.
- 구독은 회원, SKU, 수량, 배송 주기와 다음 주문 예정일을 설명한다.
- 회원은 자신의 구독과 다음 주문 예정일을 확인한다.

이 관계는 제품 개념 사이의 관계다. ERD 카디널리티, 데이터베이스 외래 키, 테이블 관계, JPA 연관관계, 객체 참조 방향으로 확정하지 않는다.

## 대표 사용자 흐름

```text
회원 로그인
→ 상품 탐색
→ 구독 가능한 SKU 선택
→ 수량과 배송 주기 선택
→ 구독 생성
→ 자신의 구독 확인
→ 다음 주문 예정일 확인
```
