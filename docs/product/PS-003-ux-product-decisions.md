# PS-003 UX 제품 결정

## 작업 목적

UX-001에서 열린 상태로 남은 네 가지 제품 정책을 사용자 승인 Product Decision으로 기록한다.

이 문서는 후속 UX-002, ARCH-001, API-001, 인증 설계와 QA가 상품 탐색, 로그인 복귀, 구독 생성 성공 이동, 다음 주문 예정일 표시 정책을 추측 없이 사용할 수 있도록 PS-002 요구사항과 연결한다.

## 선행 입력

- `docs/product/PS-002-first-mvp-requirements.md`
- `docs/domain/DOMAIN-001-first-mvp-subscription-domain.md`
- `docs/design/UX-001-first-mvp-subscription-experience.md`
- `docs/handoffs/UX-001/ux-to-tl.md`
- PR #11 `docs(ux): 1차 MVP 구독 사용자 흐름 설계`

## 결정 상태 기준

Approved Product Decision:

- 현재 사용자가 승인한 네 가지 결정이다.
- PS-002의 요구사항과 인수 조건에 반영한다.
- 후속 UX와 엔지니어링 역할은 이 결정을 제품 입력으로 사용한다.

Deferred Technical Decision:

- 구체 URI, HTTP 상태, 오류 코드, 응답 JSON은 API-001에서 결정한다.
- Spring Security 구현과 프론트엔드 라우팅 구현 방식은 후속 인증 설계 또는 FE 구현에서 결정한다.
- 로그인 복귀 경로 저장 방식과 Open Redirect 방지 코드는 기술 설계에서 결정한다.

## 승인된 네 가지 Product Decision

### 결정 1. 비회원 상품 탐색

상태: Approved Product Decision

- 비회원은 상품 목록을 조회할 수 있다.
- 비회원은 상품 상세와 SKU 정보를 조회할 수 있다.
- 비회원은 SKU의 구독 가능 여부를 확인할 수 있다.
- 구독 생성은 로그인한 회원만 수행할 수 있다.
- 내 구독 목록과 내 구독 상세는 로그인한 회원만 조회할 수 있다.

공개 기능:

- 상품 목록
- 상품 상세
- SKU와 구독 가능 여부 확인

로그인 필요:

- 구독 생성
- 내 구독 목록
- 내 구독 상세

### 결정 2. 로그인 성공 후 복귀

상태: Approved Product Decision

- 비회원이 보호된 기능에 접근해 로그인 화면으로 이동한 경우, 로그인 성공 후 안전한 서비스 내부 GET 경로로 복귀한다.
- 복귀할 유효한 내부 경로가 없으면 상품 목록으로 이동한다.
- 외부 URL과 검증되지 않은 임의 경로로 이동하지 않는다.
- 첫 번째 MVP에서는 입력 폼 상태를 자동 복원하지 않는다.
- 이전 POST 요청을 자동으로 다시 실행하지 않는다.

### 결정 3. 구독 생성 성공 후 이동

상태: Approved Product Decision

- 구독 생성 성공 후 생성된 구독 상세 화면으로 이동한다.
- 구독 상세 화면에서 생성 성공 안내를 표시한다.
- 생성 직후 구독 ID와 다음 주문 예정일을 확인할 수 있어야 한다.
- 별도의 영구적인 구독 생성 성공 페이지는 만들지 않는다.
- API 응답에는 후속 상세 이동에 필요한 구독 식별자가 포함돼야 한다.
- 정확한 응답 구조와 HTTP 상태는 API-001에서 결정한다.

### 결정 4. 생성 전 다음 주문 예정일

상태: Approved Product Decision

- 구독 생성 전에는 정확한 다음 주문 예정일 날짜를 표시하지 않는다.
- 입력 검토 단계에는 선택한 배송 주기와 다음 안내만 표시한다.

```text
다음 주문 예정일은 구독을 만든 뒤 확인할 수 있습니다.
```

- 구독 생성 성공 후 서버가 계산한 다음 주문 예정일을 표시한다.
- 내 구독 목록과 상세에서도 서버가 확정한 다음 주문 예정일을 표시한다.
- 다음 주문 예정일 계산 책임은 DOMAIN-001의 승인 설계를 유지한다.
- 사용자 화면 날짜 형식은 `YYYY. M. D.`를 유지한다.
- 다음 배송 예정일은 표시하지 않는다.

## 선택한 이유

| 결정 | 선택한 이유 |
| --- | --- |
| 비회원 상품 탐색 | 상품 선택 전 정보 탐색을 열어 두면서, 구독 생성과 개인 구독 조회는 회원 정보 보호 범위로 유지할 수 있다. |
| 로그인 성공 후 복귀 | 사용자가 보호 기능 접근 중 로그인해도 원래 맥락을 잃지 않게 하며, 안전한 내부 GET 경로로 한정해 보안 위험을 낮춘다. |
| 구독 생성 성공 후 이동 | 생성된 구독의 실제 상태와 다음 주문 예정일을 바로 확인하게 하며, 별도 성공 페이지 유지 비용을 만들지 않는다. |
| 생성 전 다음 주문 예정일 미표시 | 날짜 계산 책임을 DOMAIN-001의 서버·도메인 책임으로 유지하고, 클라이언트 사전 계산과 확정 값 불일치 위험을 줄인다. |

## 검토한 대안

| --- | --- | --- |

| 로그인 성공 후 복귀 | 로그인 후 항상 상품 목록으로 이동 | 보호 기능 접근 맥락을 잃어 사용자가 다시 탐색해야 한다. |
| 로그인 성공 후 복귀 | 외부 URL 또는 임의 경로 복귀 허용 | Open Redirect와 검증되지 않은 이동 위험이 있다. |
| 로그인 성공 후 복귀 | 입력 폼 상태와 이전 POST 요청 자동 복원 | 첫 번째 MVP 범위를 넘고, 중복 구독 생성과 보안·멱등성 설계가 필요하다. |
| 구독 생성 성공 후 이동 | 별도 영구 성공 페이지 제공 | 구독 상세와 중복되는 화면을 만들고 후속 라우팅·QA 범위를 늘린다. |
| 생성 전 다음 주문 예정일 | 클라이언트에서 예정일을 미리 계산해 표시 | 날짜 기준, 시간대, 보정 정책과 서버 확정 값 불일치 위험이 있다. |

## 사용자 흐름에 미치는 영향

- 비회원과 로그인 회원은 상품 목록과 상품 상세에서 SKU와 구독 가능 여부를 확인할 수 있다.
- 비회원이 구독 생성, 내 구독 목록 또는 내 구독 상세에 접근하면 로그인 화면으로 이동한다.
- 로그인 성공 후 안전한 내부 GET 복귀 경로가 있으면 원래 보호 화면으로 돌아가고, 유효한 경로가 없으면 상품 목록으로 이동한다.
- 로그인 후 입력 폼 상태를 자동 복원하지 않고 이전 POST 요청을 자동 재실행하지 않는다.
- 구독 생성 성공 후 생성된 구독 상세 화면에서 생성 성공 안내, 구독 ID와 다음 주문 예정일을 확인한다.
- 구독 생성 전 입력 검토 단계에서는 선택한 배송 주기와 생성 후 확인 안내만 확인한다.

## 위험과 제한

- 로그인 복귀 경로 저장 방식과 검증 방식은 아직 Deferred Technical Decision이다.
- 구체 URI와 라우트 문자열이 확정되지 않았으므로 UX-002와 FE는 개념 경로만 사용해야 한다.
- API 응답 구조와 HTTP 상태가 확정되지 않았으므로 이 문서는 응답 JSON을 정의하지 않는다.
- 입력 폼 자동 복원과 이전 POST 자동 재실행은 첫 번째 MVP에서 제외하므로, 로그인 후 사용자는 필요한 입력을 다시 수행할 수 있다.
- 클라이언트가 다음 주문 예정일을 사전 계산하지 않으므로 생성 전에는 정확한 날짜를 볼 수 없다.
