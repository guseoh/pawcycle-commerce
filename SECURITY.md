# Security Policy

## 현재 상태

PawCycle Commerce는 학습·개발 단계의 저장소다. 현재는 제품·도메인·협업 기반을 구성하는 단계이며 개인정보와 실제 결제 정보는 저장하지 않는다.

## 지원 범위

현재 기준 브랜치는 `main`이다. 보안 관련 수정은 최신 `main`에서 역할 브랜치를 만들어 Pull Request로 반영한다.

## 취약점과 Secret 보고

Secret, 취약점, Webhook URL, Token, 비밀번호, 개인 키, 인증서, 개인정보, 실제 결제 정보는 공개 Issue에 작성하지 않는다.

GitHub Private Vulnerability Reporting을 사용할 수 있으면 우선 사용한다. 기능이 비활성화된 경우에도 민감 정보를 공개하지 말고 GitHub를 통해 저장소 소유자에게 먼저 연락한다.

개인 이메일 주소나 외부 연락처는 이 문서에서 추측하지 않는다.

## Secret 노출 대응

Webhook, Token, 비밀번호, 개인 키가 노출됐다고 의심되면 즉시 다음을 수행한다.

1. 노출된 값을 폐기한다.
2. 새 값을 재발급한다.
3. 영향 범위를 확인한다.
4. 필요한 경우 Git 기록 정리 방식을 별도 보안 작업으로 결정한다.

`.gitignore` 추가만으로 이미 커밋된 Secret은 제거되지 않는다.

## 보고에 포함할 정보

- 발견한 문제 요약
- 재현 조건
- 영향받는 파일 또는 기능
- 실제 Secret 값을 제거한 증거
- 예상 영향
- 이미 수행한 완화 조치

## 응답 정책

현재 저장소는 개인 학습·개발 단계이므로 공식 응답 시간은 보장하지 않는다. 보안 이슈는 일반 기능 요청보다 우선 검토한다.
